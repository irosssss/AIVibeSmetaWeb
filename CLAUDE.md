# CLAUDE.md — как действовать в этом репо

@SOUL.md

## Стек и инварианты

- **React 18 + Vite.** Точка входа `web/main.jsx` импортирует модули по порядку
  как side-effect; **порядок КРИТИЧЕН**: `vendor-globals` → шрифты → data/lib-слой
  (`mock` → `project-data` → `pdf` → `xlsx` → `ffe` → `portal-api` → `clipper` →
  `web-analogs` → `lottie-assets`) → компоненты (`ui.jsx` первым) → `app.jsx` последним.
- **Общение файлов — только через `window.*`-глобалы** (`LedgerAPI` в mock.js,
  `LedgerFFE` в ffe.js, `LedgerClipper`, `LedgerPDF`, `LedgerXLSX`). ESM-импорты — только
  в `main.jsx` и `vendor-globals.js`. Top-level функция чужого файла видна ТОЛЬКО при
  явном `window.X =` в том файле; рантайм-патч `window.X` не долетает до уже загруженных
  файлов, звавших `X` без `window.` — проверяй видимый эффект, не факт вызова.
- **Бэкенда нет.** Хранение — `localStorage` (ключи `aivibe_*` — осознанно не переименованы);
  целевой прод — Yandex Cloud Functions + YDB, точки замены помечены `// → API:`.
- **Модель денег:** две цены (себестоимость / цена клиента); наценка — только на товары,
  услуги и сборы прозрачно поверх; запас (wastePct) оплачивает клиент.
- **Роутинг** — hash (`parseRoute`/`setRoute`), публичные страницы в `PUBLIC_VIEWS`
  (портал, ченджлог, журнал, политика, оферта).

## Команды

- `npm run dev` · `npm run build` · `npm test` (Vitest). Скрипта lint нет.
- Превью: launch.json `aivibe-web`, порт **8753** — только `preview_*`-инструменты, не Bash.

## Рабочий процесс

1. **Перед началом** сверься с журналом `docs/WEB_SMETA_ROADMAP.md` (новейшие вехи —
   сразу после шапки таблицы) — чтобы не строить уже существующее.
2. **Правки** — в стиле соседнего кода; новый компонент добавляй в `main.jsx`
   в правильное место порядка и выставляй в `window`.
3. **Проверяй сам:** `npm test` зелёный + живая проверка в браузере для UI-правок
   (не проси пользователя проверить). После серии правок одного файла HMR может
   портиться — перезапусти dev-сервер. После проверок возвращай demo-данные в исходное.
4. **Фиксируй:** веха в журнал `WEB_SMETA_ROADMAP.md` (строка после `|---|---|---|`),
   относительные даты — в абсолютные.
5. **Git:** работа на feature-ветке → PR → мерж только по слову владельца («мержи»).
   Коммиты и PR — на русском.

## Ловушки (провереные кровью)

- **Браузерная проверка:** ax-дерево (`read_page`) может врать про value/checked —
  сверяйся `javascript_tool` через `document.querySelector(...).value`; клики по
  радио/чекбоксам — `el.click()`, не координаты; клик и чтение DOM в один тик видят
  СТАРЫЙ рендер (React батчит) — мерить следующим вызовом.
- **React 18 не прокидывает JSX-проп `inert`** — только ref + `el.inert = true`.
- **«Лист-на-канвасе»:** `.pd-overlay` z-150, канвас-подложка z-149, сайдбар z-160;
  смета при открытых подвьюхах прячется `display:none` (НЕ размонтируется) — чтобы
  пережили несохранённые правки.
- **Паттерны данных:** связь по имени (style/supplierMatch — строка-имя против реестра);
  снимок при публикации (портал-шара замораживает style/studioName на момент создания).
