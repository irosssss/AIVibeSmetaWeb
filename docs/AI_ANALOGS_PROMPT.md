# AI-подбор аналогов по вебу — промпт и план интеграции (п.19-2)

Статус: **каркас, выключен.** Код контура — `web/web-analogs.js` (`window.LedgerWebAnalogs`).
Блокер включения — тот же, что у этапа 2 клиппера и портала: живой LLM-эндпоинт (ключи +
домен Cloud Function) **и** источник кандидатов (веб-поиск/браузинг или фид фабрик).

## Зачем

Сейчас замены отклонённой клиентом позиции подбираются только из уже собранного дизайнером:
библиотека студии + позиции прошлых проектов (`LedgerFFE.suggestAlternatives`, скоринг по
разделу/названию/цене/габаритам). Это `AlternativesModal` в `web/components/project-detail.jsx`.

У Programa аналоги ищет ИИ **по всему вебу**: берёт карточку товара + комментарий клиента
(«дешевле», «другой цвет», «без такой глубины») и предлагает похожие позиции с ценой и ссылкой.
Наш РФ-каскад клиппера мог бы это питать — тот же провайдер-агностик (YandexGPT/GigaChat).

## Промпт

Живёт в коде как `ANALOG_SYSTEM` (системная инструкция) + `buildAnalogPrompt(target, clientNote)`
(пользовательская часть) — чтобы дорабатывать и юнит-тестировать без сети. Ключевые решения:

- **Строгий JSON** на выходе (`{"analogs":[{title,supplier,price,url,reason}]}`) — парсим машинно,
  как слой B клиппера, а не текст.
- **Слабый источник**: модель НЕ выдумывает точную цену/ссылку — при неуверенности поле пустое
  (`""`/`null`). Лучше пусто, чем враньё (клиенту нельзя показать выдуманную ссылку).
- **Буквальный учёт комментария клиента**: «дешевле» → ниже цена, «другой цвет/материал» → меняем
  именно это, «компактнее» → меньше габарит. `reason` (до 12 слов) объясняет, чем аналог отвечает
  на пожелание.
- Не повторять исходный товар.

## Контур API (уже в коде, дремлет)

```js
// web-analogs.js — транспорт-агностично, РОВНО как слой B клиппера (opts.llm инъектируется):
const res = await LedgerWebAnalogs.suggestWebAnalogs(target, {
  clientNote: "нужно дешевле",
  llm: async (prompt, ctx) => { /* вызов YandexGPT/GigaChat, вернуть JSON-строку */ },
  limit: 5,
});
// без opts.llm → { configured:false, analogs:[] } — ничего не зовёт, не бросает.
```

`suggestWebAnalogs` никогда не бросает: сбой/таймаут LLM → `{ configured:true, analogs:[], error }`.

## Что нужно, чтобы включить

1. **LLM-транспорт**: Cloud Function-прокси к YandexGPT/GigaChat (ключи в секретах, не в клиенте) —
   тот же домен/деплой, что ждёт этап 2 клиппера и портал. Аккаунт Yandex Cloud владельца — блокер.
2. **Источник кандидатов**: модель без веб-доступа аналоги «придумает». Нужен либо веб-поиск/браузинг
   на стороне функции (подать модели реальные карточки), либо фид фабрик (`feedSku`, мостик в
   `blankProduct`). До этого `price`/`url` будут пустыми у большинства аналогов — это честно, но
   слабо.
3. **Вклейка в UI**: `AlternativesModal` — после локального `suggestAlternatives` показать раздел
   «Из веба (ИИ)» с карточками `_src: "веб (ИИ)"`; «Добавить по ссылке» уже есть как ручной путь.
   `reason` рендерить как подпись-обоснование под карточкой.

## Критерии готовности (когда появятся ключи/домен)

- Бенчмарк на N реальных «отказ клиента → нужен аналог» кейсах: доля предложений с валидной
  ссылкой и релевантной ценой.
- Инвариант «слабый источник»: аналог без уверенной цены помечен, не встаёт в смету автоматически.
- Тайминг/стоимость вызова в бюджете (как этап 2 клиппера).
