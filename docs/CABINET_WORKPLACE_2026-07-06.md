# Кабинет как рабочее место дизайнера (06.07.2026)

> Исследование по заданию владельца: боли дизайнеров с форумов/сообществ (RU + EN),
> бенчмарк 14 инструментов-аналогов, карта нашего кабинета как есть → два концепта
> («Планёрка комплектатора» — петля дня; «Стол комплектатора» — боль→документ) →
> адверсариальная критика → синтез. Прогон: 6 агентов.
> Методологическая честность: полные тексты части тредов недоступны для фетча —
> опора на пересказы в поисковой выдаче; confidence у каждой боли проставлен честно.

## 1. Что мешает дизайнерам (боли, по confidence)

| Боль | Conf. | Суть |
|---|---|---|
| **Excel разваливается** | высокая | Спецификация в одном месте, согласование в почте, бюджет в третьем, трекинг доставки «не существует нигде»; «в папке три устаревшие версии»; одна сломанная формула искажает итог; консенсусный порог обоих рынков: таблица ломается на **~100 позициях / 2-3 параллельных проектах** |
| **Наценка = «откаты»** (острейшая RU-тема) | высокая | Салоны платят дизайнеру 10–25% (свет до 30%), клиент видит скидку 1–3%; часть салонов закладывает процент дизайнера В ЦЕНУ клиента; клиентские медиа массово учат «распознавать откаты» → любой процент токсичен по умолчанию; вскрытие закупочной цены = скандал |
| **Бесконечные правки без фиксации версий** | высокая | «Вносили правки всем двором»; «никто не уверен, какую версию клиент реально согласовал»; спасаются лимитом итераций в договоре и пунктом автоприёмки через N дней |
| Клиент покупает сам по подборке / торгуется «нашёл дешевле» | средняя | Уносит ссылки и артикулы, дизайнер теряет и агентские, и оплату комплектации; спасаются: подборка без артикулов до оплаты услуги, фикс за комплектацию, предоплата |
| Срывы поставок, брак, замены снятых позиций | средняя | Клиент злится на дизайнера за чужие сроки; замена аналога тянет пересогласование; спасаются ролью «комплектатора» с графиком поставок |
| Смета устаревает быстрее таблицы | средняя | В активной закупке апдейты нужны ежедневно; RU-волатильность: +4–22%/год по регионам |
| Работа без предоплаты/договора | средняя | «Дизайн не понравился — не заплачу»; закупка на деньги клиента без агентского договора — юр-риск |
| Хаос мессенджеров; салоны не платят агентские | низкая | На низкой confidence приоритет не строим |

**Что дизайнерам помогает уже сейчас (работающие привычки):** готовые шаблоны FF&E-таблиц
(их активно ПОКУПАЮТ — структура таблицы сама по себе ценность); агентский договор + отчёт
агента; делёжка скидки салона (3–5% клиенту, 5–7% себе); предоплата 50–100%; лимит бесплатных
итераций; фикс за комплектацию вместо процента; правки одним пакетом-документом.

## 2. Что уже есть на рынке и работает (бенчмарк 14 инструментов)

Запад зрелый (Programa/DesignFiles/Mydoma $49–70 соло; Studio Designer/Design Manager —
бухгалтерия студий; Fohlio/Gather — enterprise ~$1000/user/год). **РФ — вакуум**: западные
в RU-обсуждениях не фигурируют (оплата/санкции), ниша занята Excel/Notion; «Кабинет Дизайнера»
прячет цены и живёт на серых бонусах поставщиков; «Приложение 101» — прораб, не FF&E; Faradise умер.

**Паттерны, проверенные рынком (переносим):**
1. **Клиппер «страница товара → позиция»** — ядро категории, есть у каждого успешного игрока, фирменная фича самых дорогих (Fohlio). Наш блокер #1 (слияние) = правильная ставка.
2. **Клиентский портал согласования по позициям** — magic-link без логина, approve/reject с комментарием и таймстемпом. Второй столп. Глюки в клиентских касаниях бьют больнее всего (анти-урок Mydoma).
3. **Две цены (закупка/клиенту) как доменная сущность** — у всех западных; в РФ-тулах отсутствует полностью. Уже сделано у нас.
4. **Статусы закупки по позициям + сроки** — удерживающая фича (Design Manager живёт на ней 40 лет); делать ПРОЩЕ их, не глубже.
5. **Продавать экономию часов**: «5–6 часов в неделю» (Gather), порог «Excel ломается на 100 позициях» — главный конверсионный месседж категории.
6. **Доверие — валюта №1.** Теряется тремя способами: конфликт интересов (Houzz тегал фото дизайнеров ссылками на дешёвые аналоги — отрасль помнит 8 лет), биллинг-ловушки (98% негатива Houzz Pro), ошибки в денежных цифрах (нули вместо цен у Fohlio). Наше «анти-Ivy» обещание: не продаём вашим клиентам, не трогаем ваши данные.
7. **Красота инструмента = имидж дизайнера** («Programa делает меня профессиональнее в глазах клиента») — warm editorial и качество экспорта «Для клиента» — не косметика.
8. **Соло-сегмент выигрывается** узким отполированным ядром + быстрыми итерациями + честным публичным прайсом (DesignFiles 4.6 против 4.2 у «всё-в-одном»).

## 3. Концепт: «Стол комплектатора» с языком петли

Синтез двух концептов по вердикту критика: дисциплина честности «Стола» + скелет петли
**Сбор → Согласование → Закупка → Сдача** из «Планёрки». Каждая боль превращается в документ,
который дизайнер не боится показать клиенту.

### IA кабинета (целевая)

- **Проекты** (главная) = пайплайн стадий петли + виджет **«Сегодня в работе»** (версии ждут ответа
  дольше N дней; закупка без движения; цены старше 30 дней; ближайшие поставки по lead time —
  всё из реальных данных, пустой виджет честно пуст).
- **Смета-документ** = ядро: редактор позиций + клиппер + версии + статусы закупки + свежесть цен.
- **Сорсинг-инбокс** (эволюция Избранного): буфер «увидел товар → положил в смету», пополняется
  клиппером, перенос — реальными позициями (ремонт двух нечестных мест текущего Избранного).
- **Мастерская** = стили + нормы под одной крышей (редакторы недельного ритма, освобождает топбар).
- **Профиль** — после чистки муляжей.

### Порядок внедрения (вердикты критика учтены)

| Шаг | Что | Усилия | Зависимость |
|---|---|---|---|
| A | **День чистки + язык петли + Мастерская**: вычистить муляжи (хардкод-цифры входа, статичная «AI-сессия», лейбл «YandexGPT», график активности, тумблеры-пустышки); статусы проектов → стадии петли (с миграцией старых из localStorage); объединить стили+нормы | 1–2 дня | нет — можно до слияния |
| B | **Модульное слияние веток** (не big-bang — ветки разошлись от init): `ffe.js` → PositionEditor → клиппер → версии → статусы закупки → ExtrasEditor; **Vitest денежной арифметики первым коммитом**; услуги/сборы согласовать с полями скидка/доставка/монтаж design-ветки (иначе два источника правды в деньгах) | месяц (честно) | роадмап #1 |
| C | **RU-дифференциаторы поверх слитого ядра**: (1) **паспорт свежести цен** — `price_date` на позиции, чип «цене N дней» (терракота >30), строка «цены актуальны на [дату]» во всех экспортах, «перепроверить по ссылке» с диффом старая/новая; (2) **режим «Превью»** — экспорт без артикулов/ссылок/поставщиков + честный чекбокс «услуга комплектации оплачена» (плашка про скидки — опционально, это обещание дизайнера, не наше); (3) **отчёт агента** с делёжкой скидки салона (доля клиенту/себе) — жанр RU-права, **которого нет ни у одного западного конкурента** | ~2 недели | после B |
| D | **Виджет «Сегодня в работе»** + **диф версий и счётчик итераций** (единственный элемент, прямо бьющий в боль бесконечных правок; + поле «лимит итераций по договору» рядом со счётчиком) | неделя | после B (события) |
| E | **Worker-портал согласования по magic-link** (read-only «Для клиента», принять/отклонить позицию с комментарием и таймстемпом; анти-Ivy-обещание в интерфейсе) — первая серверная фича, замыкает стадию Согласование; до него согласование фиксируется ручными статусами версий | месяц | после B и D; Worker общий с LLM-планом |
| — | «Планёрка» как полноэкранная главная — эволюция виджета при реальной плотности событий (после E), не раньше — иначе пустая витрина | — | после E |

Отложено: переключатель моделей монетизации проекта (наценка/агентские/фикс) — трогает
инвариант UI=PDF=Excel, вернуться по живому спросу; «приложение к договору» с реквизитами
и подписями — граница юр-конструктора, взять только поле лимита итераций.

### Kill list (объединённый, сверен с анти-уроками рынка)

Чат с клиентом внутри кабинета; бухгалтерия/инвойсинг/гроссбух (анти-урок Studio Designer);
маркетплейс/каталог фабрик на старте (урок Faradise: сначала привычка к клипперу, фид потом);
юр-конструктор договоров; смета ремонта/стройки (территория Remplanner/101); реалтайм-трекинг
доставки с интеграциями ТК (статусы ручные и честные); реестры покупок в чужих салонах;
клиентский портал **с логином/аккаунтами** (magic-link без логина — можно и нужно).

## 4. Слепые зоны (зафиксировать, решить отдельно)

1. **Монетизация не связана с петлёй**: канон тарифов — кредиты РЕНДЕРОВ, а ценность кабинета —
   клиппер/согласование/статусы, где рендеры не фигурируют. Бенчмарк-альтернативы: за активный
   проект (Teroom), за место ($50–70 запад), freemium-порог (101). 🔴 Решение владельца.
2. **Мобильный сценарий**: ссылки приходят в WhatsApp на телефон, статусы обновляются из салона,
   клиент откроет magic-link со смартфона. Слабый мобайл — жалоба №1 на лидера (Programa).
3. **Скорость интерфейса как фича** (анти-урок Houzz Pro «минута загрузки при клиенте») —
   ещё один аргумент за Vite-сборку (роадмап #2).
4. **Замер «смета на 100 позиций за X минут вместо вечера в Excel»** — главный месседж категории,
   заложить в воронку после слияния.
5. **Публичный честный прайс** — оба RU-конкурента прячут цены; биллинг-ловушки = 98% негатива
   Houzz. Прозрачный до скуки биллинг — часть отстройки.
