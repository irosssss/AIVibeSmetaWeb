# Дизайн-аудит UI/UX по эталону Programa.design — 2026-07-08

Запрос владельца: «везде ли у нас правильный шрифт и размер? везде правильные дашборды? везде ли правильная анимация и где её можно было реализовать? + разобрать подход Programa к дизайну сайта».

**Метод:** многоагентный воркфлоу (12 агентов): 3 агента разобрали живой programa.design (реальные CSS/JS-бандлы через curl, не скриншоты), 4 аудировали наш код по осям типографика/дашборды/анимации/лендинг, каждая находка прошла адверсариальную верификацию по file:line, в конце — критик полноты (нашёл 19 пропусков и 3 фактических неточности аудитов). Итог: **80 подтверждённых находок**, 4 отклонены как вкусовщина/ложные (сохранены в §7).

Связь с роадмапом: продолжает `PROGRAMA_BENCHMARK_2026-07-08.md` (там — фичи, здесь — дизайн-язык) и `DESIGN_AUDIT_UPROCK_2026-07-01.md`. Волны внедрения — §6.

## 0. Вердикт одной строкой

Канон warm editorial подтверждён эталоном почти буквально (их палитра #fcfbf9→#1a1916 ≈ наша бумага/чернила, их триада шрифтов = наша) — **направление верное, менять его не надо**; но у нас системный слой врёт (мёртвые токены шкалы/отступов при живой ad-hoc-рукописи), деньги местами набраны Spectral вопреки собственному правилу «цифры = mono», дашборды держатся на дублях вместо примитивов, промо анимировано богато, а рабочее ядро (смета, портал) — почти нет, и первое касание бренда вне сайта сломано (OG 1600×2000 вместо 1200×630, «aivibe» в именах клиентских файлов).

Ответы на три вопроса владельца:
- **Шрифты и размеры — НЕ везде.** Семейства дисциплинированы (все fontFamily через `var(--font-*)`), но шкала размеров живёт мимо токенов: `--fs-100..900/--lh-*/--tr-*/--sp-1..12` объявлены и **ни разу не используются**, фактическая шкала — ~29 рукописных значений с полупиксельными сиротами (12.5px — 82 вхождения, 14.5 — 51, 13.5 — 40…). Деньги в Spectral в 5+ местах. Inter 800 не загружен, но используется (фальш-жирный). PDF — чужая холодная палитра старого бренда.
- **Дашборды — НЕ везде.** База крепкая (токены, Modal с focus-trap, словари статусов ffe.js), но 2 high-контраст-бага (FavCard тёмное-на-тёмном; чип версии красит текст заливочными токенами), KPI/чарты/пустые состояния/поиск/шапки экранов продублированы между кабинетом и админкой в разных вариантах, портал типографически разошёлся со сметой.
- **Анимации — правильные там, где есть, но их нет там, где живёт продукт.** Инфраструктура выше средней (Lottie-обёртка образцовая, reduce-override, компоситные ховеры), дефектов 7 (главный: JS-скроллы `behavior:'smooth'` обходят prefers-reduced-motion; reduce не гасит transition; мёртвый hero-Lottie = 36% веса lottie-assets.js). Где добавить — 9 адресных мест (§5), все в духе «сдержанное издательское».

## 3. Разбор Programa: анимации и микро-взаимодействия

Главный вывод: **эталонный премиум-сайт индустрии обходится без GSAP/Lottie/AOS** — CSS keyframes + inline-стаггер + один IntersectionObserver-хук + точечный framer-motion. Наш стек (React+CSS+Lottie) достаточен и даже богаче.

| Паттерн | Конкретика | Вердикт |
|---|---|---|
| Бенто-хореография hero | въезд карточек fade+translateY(−6px) .8s ease-out, inline-delay 0–1.8s → вечный бобинг 7–9s амплитудой 1–3px (разные длительности ломают синхрон) | **Взять** для плиты сметы: строки/бейджи въезжают каскадом, затем еле заметный бобинг ≤3px (не мешает чтению цифр) |
| Скролл-параллакс без JS | `@supports (animation-timeline: view())` — заголовок ±6%, фон ±10%, только desktop, без JS-фолбэка | **Взять как есть**: разъезд Spectral-заголовка и «листа сметы» на ±4–6%; двигать только transform (не грейн-фон) |
| Reveal-on-scroll | IO-хук {threshold .12, rootMargin −6%, once}; скрыто translateY(8px)+opacity:0 → 450ms ease-out; при reduce контент виден сразу | **Взять**: 8px-подъём «издательнее» типовых 30px; обязательно — при reduce не прятать контент |
| Каскадная микро-сцена (3 такта) | бейдж .6s → текст .5s delay .65s → стрелка .4s delay 1.15s | Рецепт промо-сцен «позиция → наценка → итог» чистым CSS — дешевле новой Lottie |
| Hover-lift | transform .48s cubic-bezier(.22,1,.36,1) + двойная drop-shadow; гейт @media (hover:hover) and (pointer:fine) and (no-preference) | Кривая = наш --ease (совпадение!). Тени пересчитать на чернильную rgba (не чёрную) — не «холодить» бумагу |
| Cursor-repel (магнит от курсора) | 6px смещение по pointermove через rAF | **НЕ брать** — противоречит канону «спокойная бумага, не шоурум» |

| Логомарквиз | 20s linear infinite, контент ×3, motion-safe: | Отложить до фида фабрик (нечего крутить) — механика 10 строк, в копилку |
| Шкала таймингов 3 яруса | контролы .15–.2s ease; меню/оверлеи .25–.3s; reveal .45–.7s ease-out | **Взять как систему** — у нас 13 разных длительностей без токенов (§5) |
| Reduced-motion тройная защита | CSS media на каждый keyframe + motion-safe: варианты + JS-флаг (duration×0, reveal сразу виден) | У нас 2 дыры: JS-скроллы и transition под reduce (§5 P0-фиксы) |
| Продукт в движении везде | Wistia-автоплей без хрома + webm-лупы в каждом блоке фич | Принцип взять, носитель — наши Lottie |

## 8. Чему Programa учит сверх фич (дополнение к §4 бенчмарка)

- **Дизайн-канон подтверждён независимо**: западный эталон индустрии пришёл к той же формуле (тёплая бумага + серif-дисплей + гротеск + mono-мета + один акцент) — направление «Atelier» не менять, углублять токенами.

- **Анимации без библиотек**: CSS + один IO-хук закрывают весь премиум-уровень; наш Lottie — не отставание, а отличие.
- **Продукт в движении в каждом блоке фич** — ни одного мёртвого скриншота.
- **Дисциплина одного врага** (Excel) и **одного акцента** (только CTA).
- **Первое касание — не сайт**: OG-превью, фавикон вкладки, имя скачанного файла — Programa безупречна именно там, где у нас нашлись дыры.
