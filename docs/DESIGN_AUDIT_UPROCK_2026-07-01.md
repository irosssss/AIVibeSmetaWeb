# Дизайн-аудит AIVibe против баз UpRock — 2026-07-01

Бенчмарк веб-сметы AIVibe против трёх баз UpRock как эталона правил:
- **baza.uprock.ru** — каталог ресурсов дизайнера (шрифты/мокапы/цвет/иконки/иллюстрации/UX-сервисы).
- **projects.uprock.ru** — галерея кейсов + таксономия стилей/тематик (интерьер, недвижимость — прямая тема) как планка «сильного кейса».
- **uprock.ru/education** — учебник UX/UI (типографика, контраст, швейцарская/русская школы, дизайн-системы, ошибки конверсии, обратная связь).

Метод: 17 агентов (4 извлекли принципы UpRock → 12 честно проверили каждый слой продукта по реальному коду → 1 синтез). Цифры контраста/подсчёты сверены с кодом вручную.

---

## Вердикт

AIVibe **уже держит планку UpRock на уровне архитектуры**: токены как single-source-of-truth (123× `var(--muted)`), правильная тройка гарнитур (Spectral + Inter + JetBrains Mono, все с кириллицей), честная модель «дельта поверх канона» в редакторе норм, заложенные `prefers-reduced-motion`/`prefers-contrast`/`:focus-visible`, образцовая Lottie-обёртка с graceful degradation. Для no-build прототипа это выше среднего.

Но продукт системно отстаёт по **трём рычагам**:

1. **Доступность цвета — дефект №1.** Почти вся приглушённая палитра проваливает WCAG 4.5:1, и именно на этих токенах живут load-bearing цифры сметы, которые клиент читает и **печатает**:
   - `--muted` = **4.13:1** (порог 4.5) · `--faint` = **2.29:1** (несёт артикулы/количества/коды) · терракота-как-текст = **3.9:1** · текст на CTA в момент hover = **2.99:1** · `--chart`-текст = **2.77:1** · `--hairline` = **1.32:1** (разделители строк сметы почти не видны).
2. **«Моно-ритм» задекларирован, но не существует как система.** 0 токенов кегля/интерлиньяжа/трекинга/отступа; **290 инлайн-`fontSize`** (~28 кеглей); 17 значений `gap`; `tabular-nums` только в промо-плите (4×), а в **рабочем ядре сметы цены набраны пропорциональным Spectral и «пляшут»**; `font-weight:800` (26×) на Spectral, который грузится только до 700 → синтетический faux-bold, уродующий кириллицу.
3. **Воронка и обратная связь.** На лендинге 0 соц-доказательства, цен, триала; вся системная связь — **9 нативных `alert/confirm/prompt`** (ломают warm-editorial тон); модалки/меню/оверлеи недоступны с клавиатуры (нет Esc, focus-trap, `role=menuitem`).

Планка award-кейса UpRock ловит ещё одно: между красивым `hero` и служебным ядром сметы есть **разрыв** — они должны читаться как один «цветовой мир».

## Что уже сделано хорошо (не ломать)

- Токены-акценты семантичны: `--accent`=CTA/варнинг, `--accent-2`=норма/успех, `--info`, `--chart` — цвет несёт смысл, не украшает.
- `--info` (navy) = 8.51:1 и `--accent-2` (olive) = 5.31:1 — читаемы и как текст, и как фон.
- `SmetaPlate` (site-hero.jsx:127) — живой регулятор наценки, работающая демонстрация ядра, а не скриншот. Лучший элемент лендинга.
- Lottie-обёртка (ui.jsx:87–147): SVG-fallback, `playOnView` через IntersectionObserver, respect reduced-motion — продуманный, не декоративный моушн.
- Хеш-роутинг переживает F5/«назад», гейт от мигания промо при регидратации сессии, SRI-хеши на CDN.
- Модель «дельта поверх канона» реализована по-настоящему (хранится только `normsOverride`), `effNorms` в редакторе = боевой вызов — обещание «эти нормы применятся в проектах» не врёт.
- Тексты движка эргономики: причина + порог + действие («Узкий проход 64 см — норма ≥ 70 см, сместите мебель»).

## Принципы UpRock (выжимка, к которой привязан аудит)

**Типографика/визуал** (`/education/gayd-po-tipografike`, `/tipografika-v-vebe`, `/kontrast-v-ux-ui-dizayne`, `/minimalizm-skuka`, `/shveycarskiy-stil`):
- Иерархия — через **разницу кегля**, не только через жирность. Тело 14–18px, заголовки 20px+.
- Интерлиньяж тела 1.3–1.5; заголовков — поджат 1.0–1.15. Длина строки ~60 знаков, флаговый набор слева.
- ≤3 гарнитур. Трекинг: плотный на крупных заголовках, разрядка на мелком капсе.
- Контраст: **мин. 4.5:1** обычный текст, 3:1 крупный — порог, ниже нельзя.
- Один доминантный акцент; переизбыток контраста = отсутствие контраста. Пустое пространство — активный элемент.
- Против «стерильности» — управляемая асимметрия, грейн, рукотворность (польская школа); крупная типографика как элемент (русская школа) — дозированно, только в промо.

**UX/психология/структура** (`дизайн-система`, `продуктовое мышление`, `ошибки конверсии`, `эмоциональные триггеры`, `лестница преимуществ`, `хлебные крошки`, `презентация по Джобсу`):
- Дизайн-система = единый источник истины; один токен — одна роль.
- Всегда давать **видимую обратную связь** о состоянии (ошибка №1 — её отсутствие). Ошибка объясняет причину И способ исправления.
- Не прятать цены; доверие — цифрами и отзывами. Первый экран: одно предложение + один CTA, суть за 5 секунд. Фокус через отказ (Джобс).
- Wayfinding: хлебные крошки при иерархии ≥3; поведение «как везде» (закон Якоба).
- Пустое состояние = подсказка-действие, не тупик.

**Планка подачи** (projects.uprock.ru): сильный кейс = сильный первый экран + крупная типографика + единый цветовой мир + модульная сетка + консистентность. Интерьер/недвижимость — прямая тематика; экран сметы, который видит клиент, — тоже кейс-экран.

## Быстрые победы (можно сделать сразу, S/M · high)

1. **CTA hover-контраст**: `--accent-strong: #A8462A`, на `:hover` затемнять фон, не осветлять. Убрать `#FCF6EE`.
2. **`--muted` → .70** (~4.7:1) — не ждать `prefers-contrast`.
3. **Системный `.btn:disabled`** — гашение glow/фона, `cursor:not-allowed`, отключить hover-transform.
4. **faux-bold**: догрузить `0,800` в Google Fonts **или** опустить дисплей на 700 через токены веса.
5. **Success-триггер** при сохранении норм/стилей (пульс «Сохранено · мой канон» с галочкой на 2–3с).
6. **`--chart-ink`** для охристого текста (заливки графиков оставить `--chart`).
7. **Адаптив редактора норм** (`1fr` на ≤880, один столбец на ≤560).
8. **LCP hero**: проп `priority` (`loading=eager`+`fetchpriority=high`) + `<link rel=preload>`.
9. **SEO/OG/favicon** в `<head>` (description, OG 1200×630, canonical, favicon из бренд-SVG, `<noscript>`).
10. **Спрятать «Админку» и мёртвые ссылки** с публичного лендинга.
11. **Helper склонения** `plural(n, [...])` на все счётчики.
12. **H1 = УТП** «Смета клиенту с вашей наценкой» / «за минуту, а не за вечер в таблицах».

## Инструментальные пробелы → ресурсы «Базы» UpRock

- **Нет проверки контраста** → *Designer Services → Подбор цвета (Huetone, OKLCH+APCA)* + *UX (Attention Insight)*. Прогнать все пары перед разводом токенов (#1).
- **Промо голыми скриншотами** → *Мокапы (Mockup Maison, Angle)* — матовый «бумажный» мокап распечатки сметы в интерьере, не глянцевый gadget-shot.
- **Пустые состояния текстовые** → *Иллюстрации (Notioly/Storyset line-art)* — тёплая line-art «пустой лист сметы», совпадает с анти-AI рукотворностью.
- **Расширение весов Spectral / альт-моно** → *Шрифты (Fontshare, Rufonts)* — только источники с **полной кириллицей** (у Fraunces/Bricolage её нет).
- **Иконки действий кабинета** → *Иконки (Iconoir, Tabler)* — один stroke-набор под все действия.
- **Дублирующая аналитика** → *Инфографика (Google Charts как язык)* — один сильный график структуры сметы вместо двух слабых.
- **Материалы-чипы плоские** → *Фото (Kaboompics)* — матовые сканы дерева/камня/ткани под `.smeta-sheet`.
- **СТОП-лист**: НЕ брать генераторы mesh-градиентов (*Градиенты → Mesh-Y*) — это ровно «ИИшный» язык, забракованный владельцем. Для фона — только SVG-грейн.
- **Доводка отступов/выравнивания** → *Игры (Can't Unsee, Shape Type)* на насмотренность.
