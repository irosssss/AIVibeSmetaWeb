## 2. Пошаговый деплой (15 минут)

1. **Собрать:** `npm run build` → папка `dist/` (в ней index.html, assets/, img/,
   robots.txt, sitemap.xml, .htaccess — три последних добавлены аудитом 16.07).
2. **Смоук-тест локально:** `cd dist && python3 -m http.server 9000` →
   http://localhost:9000 — лендинг и кабинет живые.
   ⚠️ `file://dist/index.html` НЕ сработает (CORS на module-скриптах) — это не дефект.
   ⚠️ На localhost включается DEV_MODE (прото-свитчер) — на проде его не будет.
3. **Залить СОДЕРЖИМОЕ dist/** (не саму папку) в
   `/home/des7755176/designledger.ru/docs`:
   проще всего — zip содержимого dist → Файловый менеджер панели → загрузить → распаковать.
   **Правило: на хостинг попадает ТОЛЬКО содержимое dist/.** Корень репо не заливать
   никогда — там `.env` с живыми ключами (в git его нет, но FTP-ошибку страховка
   в .htaccess закрывает: dotfiles не отдаются).
4. **Включить SSL:** панель → Сайты → designledger.ru → Безопасность → сертификат
   (бесплатный Let's Encrypt). После активации HTTP сам редиректится на HTTPS
   (правило уже в нашем .htaccess).
5. **Проверить:** https://designledger.ru — лендинг; /robots.txt отдаётся;
   «замочек» в браузере; несуществующий путь /foo открывает приложение.
6. **Обновления дальше:** правка → `npm run build` → перезалить содержимое dist/.
   index.html не кэшируется (no-cache в .htaccess), ассеты хэшированы — пользователи
   получают новую версию сразу.

## 3. Блокеры до открытия домена (3)

### Б1. OAuth-заглушка имитирует успех — единственная нечестная заглушка
`web/mock.js:218` — «Войти через Яндекс ID / VK ID» без OAuth логинит любого
посетителя демо-персоной («Ирина Соколова», **role=admin** → бейдж «Администратор»;
VK → «Максим Орлов»). E-mail-форма при этом честно говорит «подключится с бэкендом»,
но… советует «Пока войдите через Яндекс или VK» — направляет на нечестный путь.
Второе устройство = свежая демо-персона без своих смет → субъективно потеря работы.
**Решение владельца:** пометить кнопки «Демо-вход / посмотреть на демо-данных»
(+ убрать совет из e-mail-формы, роль admin → user) ЛИБО отключить кнопки до бэкенда.

### Б2. Юр-реквизиты: 7 полей [УКАЖИТЕ …] (152-ФЗ)
`web/components/legal-pages.jsx` — заполняет ТОЛЬКО владелец (граница SOUL):
- стр. 23 — оператор ПДн: наименование ИП/ООО, ИНН, ОГРН/ОГРНИП, адрес **+** e-mail оператора;
- стр. 81 — e-mail для запросов субъектов ПДн;
- стр. 103 — оферта: реквизиты Исполнителя (наименование, ИНН, ОГРН, адрес, e-mail);
- стр. 126 — платёжный провайдер (можно оставить до подключения оплаты — оплата честно выключена);
- стр. 130 — e-mail для возвратов;
- стр. 159 — полные реквизиты вкл. банковские.
После заполнения поднять `version` с «0.9 (черновик)» — плашка-дисклеймер снимется сама
(гейт `draft = version.includes("черновик")`, стр. 185).

### Б3. Бэкдор ?dev=1 — ✅ ЗАКРЫТ 16.07 (решение владельца «убирай»)
`web/components/cabinet.jsx:59` — DEV_MODE = localhost **ИЛИ `?dev=1`**. На проде
`designledger.ru/?dev=1#admin` открывал бы админку любому (авто-эскалация в фейкового
админа). Query-ветка убрана, DEV_MODE = только localhost/127.0.0.1.
Проверено на прод-симуляции (сборка на не-localhost хосте): `?dev=1#admin` → лендинг.
