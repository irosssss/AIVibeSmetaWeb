> Задача владельца: «такой же функционал, переписанный под нас». Programa — западный эталон
> FF&E-платформы для дизайнеров интерьера (6000+ студий, 80+ стран, $71/мес за место).
> Это ровно наш жанр: спецификация/комплектация, НЕ смета ремонта.
> Связанные документы: `WEB_SMETA_ROADMAP.md` (план), `CABINET_WORKPLACE_2026-07-06.md`

## 0. Вердикт одной строкой

Ядро Programa (спецификация → цены → согласование → закупка → шаринг) мы уже во многом
построили независимо, а эргономика+оптимизатор — наш ров, которого у Programa НЕТ.
Главные дыры у нас: **клиентский портал**, **библиотека товаров студии**, **комментарии/протокол
согласования**, **счета клиенту**. Именно они делают Programa «платформой», а не табличкой.

## 1. Устройство Programa (карта продукта)

**Позиционирование:** «единственный PM-софт, построенный вокруг того, как реально работают
дизайнеры» — заменяет 4–6 разрозненных инструментов. Обещание: 6–10 часов экономии на проект.

**Модель данных:** Проект → несколько schedules (спецификаций) → секции/комнаты → позиции.
Плюс сквозные сущности: Product Library (студийная), Image Library, клиентский дашборд на проект.

**Воронка продукта — 6 шагов** (используется и как нарратив лендинга):
`01 Capture → 02 Price → 03 Approve → 04 Procure → 05 Share → 06 Connect`

### Модули

| Модуль | Суть | Ключевые детали |
|---|---|---|
| **Schedules/Спецификации** (ядро) | Позиции по комнатам/секциям | Любые поля + неограниченные кастомные; RRP и закупочная цена; ввод: клиппер / библиотека / вставка URL / вручную / Excel-импорт («Smart Importer»); drag-and-drop, bulk-операции; детект дублей кодов; тумблеры видимости полей по аудитории (скрыть цены/поставщика/бренд для клиента) |
| **Product Library** | Студийный каталог проверенных товаров | Поля: SKU, бренд, поставщик, lead time, URL, RRP+trade price, фото (много), вложения (спеки PDF, гарантии); теги, категории, архив; «в смету одним кликом»; правки в смете НЕ трогают мастер-запись; обновил цену в библиотеке → новые сметы берут свежую; права на редактирование — у админов |
| **Web Clipper** | Расширение Chrome/Edge | Smart Fill: имя, цена, до 4 фото, URL, описание, файлы; ручной режим «выделил текст на странице → назначил в поле» (габариты, материал, lead time); клип → в смету, в библиотеку или в обе; теги на лету |
| **Client Dashboard** | Портал клиента по ссылке, без логина | Брендинг студии (лого, цвета, обложка); живые данные (не вложения); тумблеры: что видно (сметы/борды/файлы/счета); статусы по позициям Approved / Needs Review / Rejected + bulk; комментарии-треды с таймстампами на позициях; экспорт протокола согласования в PDF; ссылку можно перевыпустить/отозвать; оплата счетов прямо в портале |
| **Procurement** | Закупка внутри спецификации | 8 дат с чекбоксами на позицию: заказ/отгрузка/доставка/монтаж + 4 платёжные (аванс клиента, аванс поставщику, остаток клиента, остаток поставщику); авто-категоризация срочности Overdue/Today/This Week/Next Week/Later; генерация PO на поставщика из позиций; трек-номера + кликабельные ссылки отслеживания + вложения; Procurement Hub — сквозной вид по всем проектам студии |
| **Invoicing & Payments** | Счета из сметы | Позиции тянутся из спецификации с фото/кол-вом/наценкой/налогом; этапные счета («аванс за столярку»); скидки на строку/счёт; Programa Pay: оплата картой 3.5% / банк 1.0%, без логина; синк Xero/QuickBooks; статусы отправлен/просмотрен/частично/оплачен |
| **Pinboards (мудборды)** | Сетка-борды по секциям | Картинки из клиппера/библиотек/Pinterest; постоянная живая ссылка; заметки на картинке; Canva-embed |
| **Presentations** | Канвас: пины товаров на планах/рендерах | Пин связан со сметой live (цена/статус подтягиваются); клиент аппрувит прямо в презентации → статус летит в спецификацию; PDF A4/A3 + QR на живую версию |
| **Project Management** | Kanban/список/таймлайн, фазы, задачи | Шаблоны фаз (Concept → Schematic → DD → CD…); тег товара в задаче; тайм-трекинг часов |

### Прайсинг и маркетинг

- **Один план Pro: $71/мес (1 место), +$31/мес за доп. место, триал 7 дней без карты.** Никаких тиров.
- Лендинг: hero → лонгрид-позиционирование → 4 фичевых блока → соцдоказательства (цифры
  6000 студий / 50000 проектов) → made-with-кейсы → пресса → CTA. Есть /changelog и
  «demo on demand» (15-мин видеотур).

## 2. Сопоставление с AIVibe: есть / нет / не берём

### Уже есть у нас (независимо построено или сильнее)

|---|---|---|

| Живое редактирование | ✅ PosEditor + позиция вручную (08.07) | |
| Approve-статусы версий | ✅ версии смет с согласованием (08.07) | у Programa — по позициям, у нас — по версии целиком (дыра, см. §3) |
| Procurement: заказ/отгрузка/доставка/монтаж | ✅ 8 стадий + даты + готовность + round-trip (08.07) | платёжных дат у нас нет |
| PO по поставщикам | ✅ режим «Закупка»: лист на поставщика Excel+PDF (07.07) | |
| Excel-импорт | ✅ round-trip импортёр | у них «Smart Importer» кривых файлов |
| URL paste | ✅ «По ссылке» (клиппер фаза 1) | LLM-слой — роадмап п.8 |
| Product Library (частично) | ✅ «взять из прошлого проекта» + 5 шаблонов комплектаций | полноценной студийной библиотеки нет |
| Калькулятор на лендинге | ✅ PayoffCalc + калькулятор площади с автогенерацией черновика | у Programa такого нет |
| **Эргономика NKBA/Neufert + бюджет-оптимизатор** | ✅ **только у нас** | у Programa отсутствует полностью — наш ров подтверждён и против западного эталона |

### Дыры у нас (то самое «взять и переделать под нас») — детали в §3

1. **Клиентский портал** — согласование по позициям, комментарии, брендинг, живая ссылка.
2. **Библиотека товаров студии** — сквозная, с мастер-записями и свежестью цен.
3. **Протокол согласования** (PDF с таймстампами) — юридически ценно для дизайнера.
4. **Платёжные даты в закупке** (авансы/остатки клиент↔поставщик) + срочность + трек-номера.
5. **Счета клиенту из сметы** (этапные, с оплатой) — после ЮKassa/ИП.
6. **Клиппер-расширение** с highlight-to-field (идея уже записана в роадмапе как замена VLM-стелса).
7. **Видимость полей по аудитории** как общий механизм (у нас — жёсткие 3 режима выгрузки).

### Сознательно НЕ берём (защита от расползания, в духе роадмапа §3)

- **PM-модуль (kanban/Gantt/задачи) и тайм-трекинг** — это «замена 4–6 тулов», стратегия
  Programa. Наша петля «Сбор → Согласование → Закупка → Сдача» уже покрывает стадии;
  полноценный PM — другой продукт и годы работы.
- **Pinboards/мудборды и Canvas-презентации** — icebox. Наш аналог движения «показать комнату,
  а не список» — RoomSpecOverlay + режим «Превью» (RU-дифференциатор, уже в плане шага C).
- **Pinterest/Canva/Xero/QuickBooks** — не наш стек; для РФ вопрос «1С/Мой склад» отдельный
  и не сейчас.
- **Мультивалюта** — не наш рынок.
- **Копия прайсинга** ($71 за место) — канон утверждён 07.07: 1490/2900/4900 по активным
  проектам, не пересматриваем. Но паттерн «один понятный план» — аргумент против усложнения тиров.

## 4. Чему учит Programa помимо фич

- **Один источник правды, живая ссылка вместо вложений** — «версия по email» объявлена врагом.
  Наш принцип для портала: ссылка не меняется, содержимое живое.
- **Поле видимости по аудитории** (клиент/подрядчик/поставщик) — обобщение нашей пары
  «закуп/клиент». Закладывать в модель экспорта, не хардкодить режимы.
- **Продукт = воронка 01–06** — карта пути и для онбординга, и для лендинга, и для приоритизации.
- **Простой прайсинг** — один план + доплата за место. Нам тиры уже утверждены, но урок:
  не плодить сущности в тарифах.

## 5. Заход 7 (13.07): живое приложение app.programa.design — UI/UX кабинета

Триал-аккаунт владельца, пройдено всё приложение: Pulse, Projects → Demo Project
(Overview / Schedule Summary+Financial / дровер позиции / Share / Get quotes /
Purchase Orders / Project Management), публичная шара (List+Grid), Product Library
(+сид демо-товаров), Address Book, Studio Management, Presentations.

### 5.1. Что увидели (факты)

- **Смета (Schedule)** — два вида: `Summary` (спецификация: фото, dims W×L×D с юнитами,
  qty, lead time, colour/finish/material, БЕЗ денег) и `Financial` (Trade price, Cost,
  Markup %, Client Price, RRP, Trade discount, Client savings %, **Net profit зелёным
  по строке**). Сверху полоса: Total client price + Total profit + глобальный тумблер
  «Add markup». Секции с счётчиком и подытогом «TOTAL (EXCL. TAX)».
- **Свои коды позиций** (`CH02`, `TB01` — префикс категории + номер), редактируемое
  поле «Doc code» — язык спецификаций дизайнеров («см. чертёж, поз. CH02»).
- **Статусная лестница строки в рабочей смете**: Internal review → Client review →
  Client approved → Ordered (цветной чип-дропдаун прямо в строке). «Ordered» стыкует
  согласование с закупкой в одну воронку.
- **Дровер позиции**: prev/next стрелки (правка строк подряд без закрытия), табы
  Summary / Financial / Attachments / Approvals (история согласования по позиции,
  пустое состояние учит: «Mark as Client review to get feedback»), мультифото-галерея
  со звёздочкой primary, Doc code, SKU, lead time, Product URL, поставщик как карточка
  (компания + контакт + email, Edit/Remove), Download PDF на позицию, автосейв «Saved».
- **Кнопки добавления в потоке таблицы** (после каждой секции): Add from URL /
  Product from library / Custom Product / Section — то же трио, что у нас, плюс секция.
- **Share** — 2 режима: Client Dashboard (по email клиента) и Publish to Web (живая
  ссылка + QR). **Тумблеры приватности, все ВЫКЛ по умолчанию**: Show supplier / brand /
  product URL / pricing / details → клиент по умолчанию видит только фото/имя/габариты/
  статус. Публичная шара: «Last Updated N minutes ago», Export to PDF у клиента,
  collapse секций, переключатель **List / Grid** (Grid = визуальная галерея по секциям).
