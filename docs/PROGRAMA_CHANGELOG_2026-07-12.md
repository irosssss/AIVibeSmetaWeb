## 1. Формат их ченджлога (образец для нашего `/changelog`)

Наш `/changelog` (волна F3) — текстовые записи по датам, вход только из футера лендинга.
У Programa — **продуктовая поверхность внутри приложения**:

1. **Вход из кабинета**: пункт «What's new» в самом низу сайдбара (рядом с «Guides»),
   отдельная страница внутри воркспейса — пользователь читает релизы, не выходя из работы.
2. **Каждая крупная запись** = дата → заголовок-стори → **hero-иллюстрация**
   (стилизованный скриншот фичи на подложке, не сырой скрин) → жирные подзаголовки-фичи
   с 1–3 короткими абзацами → секция **«Where to find it»** (где включить/найти) →
   ссылки Help Center / поддержка.
3. **Сторителлинг от боли, не от фичи**: «Client pushed the install date? Joinery running
   three weeks late?» → и только потом что сделали. Ровно наш приём «боль → решение» с лендинга.
4. **Два калибра записей**: большие релизы (одна фича = один лонгрид с картинкой) и
   «Recently added features and functionality» — плоский список мелочей 1-2 предложениями.
5. **Бейдж «Coming soon»** на hero-картинке — анонсы до релиза, тизер роадмапа
   (запись 19.06 «The next chapter of Programa»: Schedules 3.0 / Studio Operations / Brands / AI).
6. Каденс ~1 запись в неделю (32 за полгода) — ритм живого продукта.

**Адаптация формата (🟢 всё можно сейчас):**
- «Что нового» в сайдбар кабинета (у нас вход только с лендинга — дизайнер в кабинете
  релизов не видит).
- Иллюстрации к записям: НЕ скриншоты, а наши же промо-мокапы как React-компоненты
  (приём SmetaPlate/ClientPortalPromo) — дешевле скриншотов и не устаревают по стилю.
- Секция «Где найти» в конце крупных записей.
- Бейдж «Скоро» для анонсов (честность: у нас уже канон «пример, не факт»).

### Кандидаты на адаптацию (наглядно показано у них, ложится на нашу петлю)
1. **Аналоги при отклонении клиентом** — их флагман двух записей (25.06 «Coming soon» +
   10.07 релиз): клиент отклонил позицию в согласовании → система предлагает замены
   из Product Library (+веб-поиск), с бейджами «56% cheaper»/«Client suggested», меню
   «Replace current product / Not interested». **У нас все кубики есть**: портал со
   статусом «Отклонено» и комментариями + библиотека товаров + прошлые проекты.
   Версия «аналоги из СВОЕЙ библиотеки» — 🟢 можно сейчас; версия из каталога фабрик —
   это наш 🔴 «Аналоги в один клик» (роадмап §2). Закрывает разрыв «клиент отклонил → а дальше?».
2. **Pulse — умная лента «что требует действия»** (февраль–март, «первая глава StudioOS»):
   решения клиента live (Approved/Rejected), просроченные платежи по инвойсам,
   **«Products missing suppliers» / «Products missing lead time»** — карточки-действия.
   У нас зачаток — «Сегодня в работе» (волна C). Расширение 🟢: события портала
   (клиент согласовал/отклонил/комментировал) + «позиции без поставщика/без стадии» как
   карточки на Столе комплектатора.
3. **Отметка «клиент открыл»** (13.03, Email tracking: Sent/Delivered/Opened/Bounced для
   инвойсов): у нас портал по ссылке — бейдж «клиент открыл портал, последний визит N»
   на карточке проекта 🟢 (мок: счётчик визитов по shareId).
4. **«Product attached to»** (13.03): позиция закупки знает свои документы (инвойсы/PO)
   со сквозным переходом. У нас аналог — позиция ↔ версии сметы/закупочный лист 🟢 (частично).
5. **Timeline переносов** (02.07): «сроки едут — это норма жанра», view что сдвинулось
   и что это тянет. У нас даты стадий закупки есть; сводный таймлайн — кандидат после D/E.
6. **Автолинки URL в комментариях** (мелочь из их списка): комменты портала у нас — тот же кейс 🟢.

### Не наш жанр (по правилу «Чего НЕ делаем» роадмапа §3)
- **Invoicing + Programa Pay + QuickBooks/Xero** — выставление счетов клиентам и
  собственная платёжка. У нас платежи = записи стадий закупки, биллинг = подписка ЮKassa.
  Не расползаться (их путь к этому — годы и регионы US/AU).
- **Time Tracking / Smart Timer, Meeting Notes (запись встреч), Presentations,
  SketchUp Extension** — студийная операционка и 3D-мост; не смета-комплектация.
  SketchUp-мост концептуально = наш отклонённый PixelRAG-класс задач.
