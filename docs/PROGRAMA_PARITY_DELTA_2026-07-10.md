## 1. Карта роутинга Programa (снята с живого приложения — раньше не фиксировалась)

**Студийный уровень** (постоянный сайдбар):

| Экран | URL |
|---|---|
| Pulse («Сегодня») | `/pulse` |
| Проекты | `/projects` · табы `?tab=all|archived` · создание `/projects/new` |
| Presentations | `/presentations` |
| Studio Management / Time Tracking / To-Do | `/studio` · `/time_entries2` · `/todo_tasks` |
| Purchase Orders (сквозной) | `/purchase_orders` |
| Product Library | `/product_library/library_products` |
| Address Book | `/address_book/company_contacts` |
| Image Library | `/image-library` |
| What's new | `/changelog` |

**Уровень проекта** (сайдбар полностью подменяется): `/projects/:id/{overview | files | meetings |
tasks | purchase_orders | shared_files | edit}`. Settings = `edit`, вкладка Client Dashboard —
внутренним состоянием (URL не меняется; у нас честнее — hash-сегмент).

**Документы проекта — на ГЛОБАЛЬНЫХ роутах, вне пути проекта**: смета `/schedules2/schedules/:id`,
пинборд `/pinboards/:id`, презентация `/presentations/:id`. Контекст проекта в сайдбаре
восстанавливается по принадлежности документа. Наша hash-схема `#cabinet/projects/{id}/{sub}`
это уже покрывает — менять нечего.

KPI-ряд Обзора линкуется вглубь: Active tasks → `tasks`, Time tracked → `/time_entries2?project=…`,
Invoices → `invoices`. (У нас так же: каждая KPI-карточка — переход, W2.2 ✅.)

## 2. Новые наблюдения (нет в §1/§5 третьего дока)

1. **Драуэр позиции держит контекст**: prev/next (▲▼ 20×20) листает позиции, НЕ сбрасывая
   активную вкладку (Financial остаётся Financial). Драуэр открывается на вкладке,
   соответствующей текущей вьюхе списка (из Financial-вьюхи → вкладка Financial).
   У нас prev/next есть (W5.3); «контекстная вкладка» — не применимо (PosEditor один).
2. **Строки сметы редактируются инлайн**: Product Description, DOC CODE, габариты, Qty — прямо
   в строке (textbox в таблице), драуэр — для остального. У нас правка только через PosEditor.
3. **Оптимистичная пересинхронизация**: после правки в драуэре строка позади показывает
   скелетон-шиммер на время refetch. Для localStorage не нужно (у нас мгновенно).
4. **Полоса итогов Financial раскрывается**: свёрнуто — Total client price + Total profit + `>`;
   раскрыто — 6 метрик (+ Client savings %, RRP, Trade price, Trade discount %) и `<` назад.
   Профит и экономия — зелёным. Тумблер Add markup справа в той же полосе.
5. **View section ▾** — меню с поиск-инпутом сверху и mono-счётчиками (спека §5.6 подтверждена);
   у нас «К комнате ▾» (W5.1) — паритет, включая поиск.
6. **⌘K**: скоуп-селектор «All Programa ▾», группа **Recently visited** (документ + «Updated N ago»,
   иконка типа), футер ↑↓/⌘↵ (new tab)/↵. Пустой инпут = недавние, не пустота.
7. **Pulse**: бейдж «Start here» на первом невыполненном шаге чек-листа; выполненный шаг
   зачёркнут с зелёной галкой (у нас так же, W3.1 ✅ — сверено бок-о-бок).
8. **Files-таблица**: Name · Type · Shared with · Status · Last updated + «Create file»;
   «Shared with»/Status пустуют до шаринга. Клик по строке = документ.
9. **Крошка `Files > Project Schedule ▾`** — каретка-кнопка рядом с именем документа,
   переключение документов проекта без выхода из вьюхи.
10. **Переходы уровней — мгновенные** (SPA, без анимаций страниц): смена сайдбара студия↔проект
    без transition, контент — скелетоны по форме будущего контента. Всё «движение» — только
    микро (меню 80–160ms, ховер-тень 150ms) — наша W5.0-физика это уже повторяет.

## 3. Сверка с нашим main (прогон 10.07): что в паритете

- Двухуровневый сайдбар с полной подменой + «← Все проекты» ✅ (W1).
- Обзор проекта: KPI живых цифр с переходами и «нулями словами», Recent-документы,
  чип статуса с дропдауном, обложка ✅ (W2). Наши цифры содержательнее их нулей.
- «Сегодня»: приветствие + дата + чек-лист 1/3 с зачёркиванием + TodayWidget ✅ (W3).
- Настройки студии (реквизиты → PDF/портал) и проекта (детали + архив) ✅ (W4).
- Физика: лист-на-канвасе, ring-тени, пилюли, меню, prev/next, EmptyState, ⌘K, N ✅ (W5).
- Студийный «Проекты» у нас БОГАЧЕ: KPI-полоса, фильтры по стадиям петли, поиск, сортировка,
  закреп (у Programa — только табы All|Archived + поиск + сортировка).

## 4. Бэклог остаточной дельты (кандидат — волна W6)

Приоритет: сначала то, что ускоряет ежедневную работу со сметой, потом навигационный сахар.

| # | Дельта | Что у Programa | Что у нас | Оценка |
|---|---|---|---|---|
| Д1 | **Инлайн-панель добавления между комнатами** | `✦ Add from URL · Product from library · Custom Product · Section` между секциями, URL-способ первый с искоркой | Кнопки добавления в тулбаре сметы, вне контекста комнаты | Средняя; идеально ляжет на клиппер («по ссылке» первым) |
| Д2 | **Переключатель документов в крошке** | `Files > Project Schedule ▾` — прыжок между документами проекта | Крошка есть, ▾-переключателя нет (только сайдбар) | Малая |
| Д3 | **⌘K: группа «Недавнее»** | Пустой запрос = Recently visited с «Updated N ago» | Пустой запрос = все проекты + разделы | Малая |
| Д4 | **Раскрытие полосы итогов** | `>` разворачивает 2 метрики → 6 (RRP, скидка, экономия клиента) | Липкий итог один; наценка — в блоке сверху | Малая-средняя |
| Д5 | **Инлайн-правка ключевых полей строки** (Qty, цена) | Textbox прямо в строке, драуэр для остального | Всё через PosEditor | Средняя; решить ПОСЛЕ живых смет (F2) — возможно, наша модель безопаснее |
| Д6 | **Чип срока проекта в шапке Обзора** | `10 Jul → 9 Dec` чипом рядом со статусом | Срок задаётся в настройках (W4), в шапке Обзора не виден | Малая |
| Д7 | **PDF позиции из драуэра** | Download PDF на одну позицию | Выгрузки только на смету целиком | Малая; ждать запроса пользователей |

**Не берём — подтверждение против расползания** (4-й раз): Get quotes, Start timer,
Quick Meeting, Meetings-AI, оргпереключатель, Image Library, Team/Shared-with, Presentations.
Скелетоны-refetch (набл. 3) — не нужны на localStorage; вернутся в тему с бэкендом.

**Визуальный канон**: warm editorial подтверждён (в 4-й раз) — физика Programa уже перенесена
в наших цветах (W5.0), рестайл «пиксель-в-пиксель» не требуется и не планируется.
