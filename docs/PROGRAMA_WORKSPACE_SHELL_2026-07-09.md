## 0. Вердикт одной строкой

У Programa продукт живёт в **двухуровневом воркспейсе** (уровень студии ↔ уровень проекта
с полной подменой сайдбара), где проект — это **контейнер документов** с обзором-дашбордом;
у нас продукт сильнее в глубину (смета/эргономика/закупка), но живёт в **четырёх вкладках
топбара**, а проект — это один полноэкранный оверлей сметы. Перенос оболочки — это каркас
W1–W5 ниже; вся начинка уже построена волнами A–C и переезжает без переписывания.

## 1. Карта воркспейса Programa (что видел на живом приложении)

### 1.1 Двухуровневая навигация — ключевой приём

**Уровень студии** (сайдбар слева, постоянный):

- Шапка: переключатель организации «Vibe ▾» (мультистудийность).
- `Search ⌘K` · `Quick Meeting`.
- Группы: **Pulse** (личный дашборд) · **Projects** · **Presentations** [бейдж New] ·
  **Studio** (⤷ Studio Management, Time Tracking, To-Do List) · **Financials**
  (⤷ Purchase Orders — сквозной по всем проектам) · **Libraries** (⤷ Product Library,
  Address Book, Image Library). Группы сворачиваются, активный пункт — карточка-«таблетка».
- Низ сайдбара: «Need help?» с **живыми аватарами команды** + зелёная точка онлайна ·
  What's new · Guides · карточка триала «Upgrade plan / 7 days left / To keep using all features».

**Уровень проекта** — сайдбар **полностью подменяется** при входе в проект:

- Шапка: `← Demo Project` (возврат на уровень студии).
- Те же Search/Quick Meeting, затем: **Overview · Files · Meetings [Beta] ·
  Project Management · Purchase Orders · Shared Files · Settings**.

**Топбар контента**: хлебная крошка `Files > Project Schedule ▾` (▾ — быстрый переключатель
между документами проекта не выходя из вьюхи), справа глобальные действия
(Start timer; в смете — Get quotes, Share).

### 1.2 Обзор проекта (Overview) — «лицо» проекта

- Заголовок-serif + мета «Melbourne, VIC · Edited 1 minute ago».
- Ряд чипов: статус (**Active ▾** — дропдаун прямо в чипе) · `+ Project type` (пустое
  значение = чип-приглашение) · `10 Jul → 9 Dec` (срок).
- Абзац описания проекта; справа — карточка-обложка (cover image).
- **Team / Clients** — два аватар-ряда с кнопками «+».
- **KPI-ряд** (большие serif-цифры, ⓘ-тултипы, стрелка `>` = переход, подпись-контекст):
  `7 > Active tasks · No overdue tasks` — `0 Pending approval · No rejected items` —
  `0h 0m > Time tracked` — `$90 022,66 Total schedule value` — `0 > Total invoices · No invoices yet`.
  Вторая строка подписи — «нулевое состояние» метрики словами, не пустота.
- **Recent** — документы проекта карточками; тумбнейл сметы — **автоколлаж из фото позиций**
  (не «иконка файла»), у презентации — тёмная обложка. Подпись: тип + «39 items».

### 1.3 Проект = контейнер документов (Files)

Таблица: Name · Type (Schedule / Pinboard / Presentation) · Shared with · Status ·
Last updated + `Create file`. Смета — не «весь проект», а один из документов проекта.
Отсюда же и Recent-карточки на Overview, и переключатель в крошке.

### 1.4 Смета (Schedule) — приёмы оболочки, не начинки

- Тумблер вьюх **Summary | Financial** + **View section ▾** — поисковый прыжок по секциям
  со счётчиками (Seating 5, Tables 4, Rugs 3, … Master Ensuite 4) — спасение длинной сметы.
- Секции с количеством и **подытогом секции** в заголовке; hover — Collapse + стрелки порядка.
- Financial: **липкая полоса итогов** «$90,022.66 Total client price · $5,184.21 Total profit ·
  [>] раскрыть» + тумблер Add markup; в строке Trade price / RRP / Qty / Trade discount / Cost /
  Client savings / Markup / **Client Price** / Net profit (профит — зелёным).
- Строка позиции: фото, тип+«See drawings for location», код CH02, имя/бренд, габариты
  с подписями-единицами под значениями, Qty, Lead time ▾, поставщик (компания + контакт),
  **статус-чип с точкой и дропдауном** (Internal review 🟠 / Client review 🟠 /
  Client approved 🟢 / Ordered 🔵), кнопки Details / Quote.
- Между секциями — **строка вставки**: `✦ Add from URL · Product from library ·
  Custom Product · Section` (все способы ввода в одном месте, в контексте).
- **Details — правый drawer** с вкладками Summary / Financial / Attachments / Approvals,
  стрелки prev/next (ходить по позициям не закрывая), Download PDF на позицию.

### 1.5 Студийный уровень

- **Pulse**: «Good evening, Mikhail» + дата, чек-лист онбординга «Getting started 1/3»
  (сделанное — зачёркнуто с зелёной галкой), «Today's focus» — пока пусто: «Smart,
  time-saving recommendations based on your projects and activity will appear here»
  + «Did you find these recommendations useful? 👍👎».
- **Projects**: сетка карточек с обложками, табы All | Archived, поиск, сортировка,
  статус-точка + «Edited N minutes ago», hover — пин-кнопка, меню «…».
  Преднасеяно: **Demo Project + 2 шаблона проектов** (Interior Design & Arch / Decor & Styling).
- **Financials → Purchase Orders**: сквозной по студии; **Libraries**: Product Library
  (пустое состояние с кнопкой **Add demo products**!), Address Book (контакты поставщиков,
  Import), Image Library.
- **Studio Management / Time Tracking / To-Do List** — PM-модуль (канбан Board/List/Timeline,
  таймер в топбаре) — мы это сознательно НЕ берём (решение в бенчмарке 08.07, подтверждаю).
- **Meetings [Beta]** — записать/загрузить встречу → AI вытаскивает решения и action items.
  Не наш скоуп сейчас; идея «протокол встречи с клиентом» — в icebox.

### 1.6 Настройки

- **Студия** (из скриншотов владельца): лого, имя, тип, локация+страна, About, Tax ID,
  сайт, соцсети; Contact details «will be displayed publicly on your studio's profile».
  Разделы: My Account / Studio / Team / Billing / Tax / Integrations / Notifications.
- **Проект → Settings**: вкладки **Details** (имя, тип, срок, обложка, описание, сайт, адрес
  с оговоркой про налоги; Archive слева внизу, Save changes справа) и **Client Dashboard**
  (обложка портала, **дефолтная вьюха сметы для клиента**, список клиентов с доступом +
  e-mail-уведомления о новых публикациях).

### 1.7 Сквозные паттерны оболочки

- **Пустые состояния продают**: иконка + один абзац ценности + CTA + «Learn more», местами
  «Add demo products». Ни одного голого «ничего нет».
- **Онбординг слоями**: демо-проект с честной пометкой в описании + шаблоны + чек-лист в Pulse
  + баннер-гайд сверху + «Need help?» с лицами + карточка триала. Продукт «заполнен» с 1-й минуты.
- Хоткеи в интерфейсе видимы: ⌘K поиск, ⌘N новая позиция.

**Что у нас лучше уже сейчас (и что оболочка должна выпятить, а не спрятать):**
эргономика-движок + оптимизатор бюджета (ров), паспорт свежести цен, живой TodayWidget,
две цены + 3 режима выгрузки, версии с протоколом согласования. «Даже лучше» = их каркас,
наполненный нашими живыми цифрами (их Today's focus — заглушка с обещанием, их KPI при

## 3. План переноса — волны W (Workspace)

Нумерация волн продолжает алфавит бенчмарка (A–F заняты). Порядок = зависимость:
W1 каркас → W2 лицо проекта → W3 студийный уровень → W4 настройки → W5 полировка.

### W1 — Каркас: двухуровневый сайдбар + крошка (фундамент, без него остальное некуда класть)

| Задача | Что именно | Критерий готовности |
|---|---|---|
| W1.1 Сайдбар студии | Вместо топбар-вкладок: Сегодня (=Pulse) · Проекты · Мастерская (⤷ Стили/Товары/Нормы) · Закупка · Избранное; низ — Что нового (`/changelog`) · профиль-карточка | Все текущие адреса `#cabinet/...` живут (redirect), топбар остаётся только на лендинге |
| W1.2 Сайдбар проекта | Вход в проект подменяет сайдбар: ← имя проекта · Обзор · Смета · Версии и согласование · Закупка · Портал клиента · Настройки | `ProjectDetail` перестаёт быть оверлеем, становится уровнем роутинга `#cabinet/projects/p_1/{sub}` |

## 5. Физика интерфейса Programa — спецификация с живого DOM (разведка 10.07, перед W5)

> Запрос владельца: «такие же отступы, иконки, анимации, логика». Все цифры сняты
> computed-стилями с app.programa.design (триал). Их нейтрали — ТЁПЛЫЕ (канвас #F8F6F3,
> чернила #1A1916, serif-дисплей Rhymes + Inter) — т.е. философски это наш же warm
> editorial. Берём физику (размеры/тени/радиусы/движение/логику), цвета и шрифты — наши
> (терракота/олива, Spectral/Inter/JBMono). Канон НЕ меняется — подтверждён третий раз.

### 5.1 Каркас окна
- Канвас всего окна — тёплый `#F8F6F3`; контент — **белый «лист»**, отступ 8px от края
  окна (сверху/справа/снизу), сайдбар слева ~220px на канвасе (без бордера — разделение цветом).
- Поля страницы внутри листа — всего **16px**; H1 — serif 24px/400 (у них Rhymes Display,
  у нас Spectral), крошки 12px/500 `#7B736C`.
- Внутри листа зоны списков снова тёплые (`#F8F6F3`, боковые поля секции 28px) —
  белые карточки-строки лежат на тёплой подложке. Трёхслойный сэндвич: тёплый → белый → тёплый.

### 5.2 Нейтрали (их значения — для сопоставления с нашими токенами)
чернила `#1A1916` · вторичный текст `#5A534C` · приглушённый `#7B736C` ·
