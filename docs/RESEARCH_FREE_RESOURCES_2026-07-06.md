# Исследование внешних репо → что берём (06.07.2026)

> Разбор двух репозиториев по заданию владельца: [cheahjs/free-llm-api-resources](https://github.com/cheahjs/free-llm-api-resources)
> (каталог бесплатных LLM API) и [BraveOPotato/FckSignups](https://github.com/BraveOPotato/FckSignups)
> (174 OSS-инструмента без регистрации). Прогон: 3 исследователя → 3 стратега
> (деньги / ценность дизайнеру / техника без бюджета, 24 идеи) → адверсариальный критик.
> Рамка прежняя: честная ценность, соло-разработчик, минимальный бюджет, НЕ маркетинг «рва».

## 0. Резюме

1. **Бесплатное топливо для LLM-слоя клиппера есть** — извлечение полей товара можно
   запустить с COGS≈0 на MVP-этапе (Mistral free / Cloudflare Workers AI / OpenRouter :free).
   Но это **мост, а не фундамент**: ToS-ограничения и лимиты «на организацию» требуют
   миграции на платные ключи с первыми платящими.
2. **Бесплатной генерации изображений интерьерного качества НЕТ** ни у кого из 26 провайдеров.
   Рендер-кредиты тарифов 1490/2900/4900 обязаны опираться на платный image-API —
   юнит-экономику в `BUSINESS_MODEL.md` пересчитать от его фактической цены (день документной работы).
3. **Главный урок FckSignups — не сервисы, а паттерн**: «ценность до регистрации».
   Гостевое демо клиппера на лендинге + «смета клиенту по ссылке без аккаунта» — это наша воронка.
4. **Слепая зона всех стратегий — дистрибуция и валидация**: ни идея не отвечает, откуда придут
   дизайнеры, и ничего не проверено на живой ЦА. Прежде чем строить месяцы фич — показать
   прототип 10 реальным дизайнерам.
5. Порядок жёсткий: **сначала слияние веток** (блокер #1 — клиппер живёт на `audit-sprint0-1-vite`),
   потом LLM-слой, потом воронка. Подключать LLM до слияния = двойная работа.

LLM только добирает пустоты после структурного разбора (JSON-LD → Microdata → OG → meta → текст),

### 🔴 Развилка для владельца: западный каскад vs РФ-канон

Роадмап фиксирует «AI — YandexGPT/GigaChat» и «полностью РФ-стек» как сильную сторону.
Западный free-каскад это ломает: ToS-серость, данные в обучение, а с появлением реальных
аккаунтов — конфликт сервера в ЕС со 152-ФЗ (локализация персданных граждан РФ).
Честная постановка: **бенчмарк обязан включать YandexGPT/GigaChat наравне с западными** —
они платные, но дешёвые в рублях, оплачиваются картой РФ и юридически чистые для прода.
Западные free-tier'ы — dev-топливо и MVP-мост; выбор прод-провайдера — по данным бенчмарка,
а не по идеологии. Решение за владельцем после таблицы «модель × точность полей × ₽/смета».

### Архитектура (выжившая после критики)

1. **Бенчмарк до каскада** (неделя): 25–30 HTML-фикстур реальных страниц РФ-магазинов
   (сохранить вручную из своего браузера — заодно узнаем фактическую долю магазинов,
   не отдающих HTML ботам) + эталонная разметка полей. Прогнать через Scaleway/Alibaba
   trial + Gemini Flash-Lite (dev) + GitHub Models GPT-4o (эталон) + **YandexGPT/GigaChat**.
   Фикстуры остаются регресс-тестами клиппера навсегда.
2. **Cloudflare Worker `/api/extract`** — первый реальный кусок бэкенда (неделя):
   ключи в Worker Secrets, JSON строго по FF&E-схеме, контракт уже размечен метками
   `// → API:` в `mock.js`/`clipper.js`. **Свой домен** (workers.dev в РФ периодически
   блокируется). Сразу — rate-limit/бюджет на IP (иначе гостевое демо сожжёт квоты).
3. **KV-кэш по нормализованному URL** (день, делать вместе с Worker): попадание в кэш =
   0 LLM-вызовов; TTL ~7 дней + **дата извлечения в UI + кнопка «обновить цену»** —
   иначе смета начнёт врать клиенту (репутационный риск сильнее технического).
4. **Каскад** primary → fallback → Workers AI; порядок фиксируется ПО бенчмарку.
   В план прямо вписана миграция на платные ключи при первых платящих
   (лимиты считаются на организацию: ~10 активных дизайнеров ≈ потолок free-квот).
5. **Анти-бот (Qrator/DDoS-Guard у РФ-ритейла)**: VLM-по-скриншоту с headless-браузера
   ОТЛОЖЕН (критик: дата-центровые IP получат ту же капчу — месяц кроличьей норы).
   Дешёвая альтернатива, не рассмотренная ни одной линзой: **клиппер-расширение браузера /
   букмарклет** — страница магазина уже открыта у дизайнера с его сессией, HTML берётся
   оттуда бесплатно и легально. Прототипировать после бенчмарка, если доля фейлов высокая.

## 2. FckSignups: паттерны и OSS

Каталог 174 браузерных OSS-инструментов без регистрации (GPL-3.0, ~854 звезды). Это не API —
интеграция = заимствование кода/библиотек. Ценность в двух слоях:

### Продуктовые паттерны (главное)

- **«Ценность до регистрации»** → гостевой клиппер-демо на лендинге: вставил ссылку →
  карточка товара + мини-смета с живой наценкой; «Сохранить/Экспорт/Рендер» → регистрация
  с переносом гостевой сметы в аккаунт. Реальный срок 2–3 недели ПОСЛЕ слияния и Worker
  (гостевой fetch из статики умрёт на CORS/анти-боте — критик подтвердил).
- **«Смета клиенту по ссылке без аккаунта»** (паттерн ente Paste/Cryptgeon) → фаза 1:
  read-only снапшот (только цены «клиенту»), QR, срок жизни — неделя работы поверх Worker+KV.
  Фаза 2: построчное «ок / заменить / комментарий» со стеканием в кабинет (ещё 1–2 недели).
  Это лучшая идея всей волны: усиливает ядро и даёт вирусный канал (внизу — «собрано в AIVibe»).
- **Local-first как честный питч**: «данные сметы — у вас в браузере, фото чистятся от
  гео-меток» — правдивый приватность-аргумент вместо маркетинга «рва».

### OSS берём

| Что | Куда | Усилия |
|---|---|---|
| Эвристики полноразмерных фото (og:image, srcset max) — идея maxurl, но БЕЗ его мегабайт правил | клиппер | час |
| Lucide (MIT) — единый набор line-art иконок под «рабочий чертёж» | вся тема Atelier | вечер |
| SVGOMG-пасс по SVG мини-планов и line-art лендинга | вес страницы | вечер |
| Squoosh (WASM-сжатие) + EXIF-скраб (гео-метка = адрес квартиры клиента!) | будущий рендер-пайплайн, когда появится реальная загрузка фото | полдня — тогда, не сейчас |
| Инфра-шаблон самого репо: канон-JSON + Cloudflare Worker для заявок + скрипты автообновления | будущий фид фабрик — приём заявок фабрик без бэкенда | по факту фида |
| Grist — подсмотреть UX inline-редактирования таблиц | смета-документ | референс |

### OSS отклонено

- **Excalidraw-редактор планов** — единственная идея с честным «месяц», и всё равно оптимизм:
  требует Vite-сборки (блокер #2 не сделан) + семантика «это стена, это диван» из freeform-фигур
  для `engine.js` — исследовательская задача. В icebox до Vite и подтверждённого спроса.
- tldraw — SDK требует лицензии/watermark для прода.
- WebLLM (LLM в браузере, обход гео-блоков) — веса гигабайты, качество мелких моделей
  под вопросом; только как любопытство.
- Cobalt (картинки из Pinterest/соцсетей) — юридически серо, не обещать.
- Stirling-PDF/BentoPDF — у нас уже есть pdfmake-конвейер, не менять работающее.

White-label (логотип дизайнера) — платным. Включать гейты одновременно с реальным биллингом, не раньше.

### Этап 4 — кабинет-ценность (не ждёт бэкенда, можно между делом)
- **«Взять из прошлого проекта»** (вердикт «брать», честный день): копирование позиций/комнат/
  разделов из прошлых смет с пометкой давности цены + свои типовые комплектации-шаблоны.
- **Закупочный лист по поставщикам** (вердикт «брать», день-два): третий режим выгрузки
  «Закупка» — группировка по поставщикам, лист на поставщика в Excel, готов приложить к заказу.
- Профили наценок «мой стандарт» (роадмап #4): реализовать открыто, поле для гейта заложить,
  лимит «1 профиль на free» включить вместе с биллингом.

### Этап 5 — деньги (когда готова бумага из этапа 0)
Реальный вход Яндекс/VK + ЮKassa. Требует серверного хранения пользователей и проектов
(миграция localStorage → сервер) — без этого подписка продаёт воздух (проекты не переживают
смену браузера). С реальными аккаунтами приходит 152-ФЗ: оферта, политика персданных,
вопрос локализации данных (конфликт с сервером в ЕС — проработать до запуска).

### Убито критиком (не делать, чтобы не расползтись)
- **BYOK** («свой ключ пользователя») — фича для разработчиков, не для дизайнеров;
  «плати подписку И сам оплачивай LLM» — антипродажа; ToS-риск перекладывается на юзера.
- **«Пульс проектов»** (событийная главная) — лента событий без событий = кабинет-театр,
  ровно то, что честная рамка запрещает. После шаринга хватит блока «требует внимания».
- **EXIF-гигиена как отдельная фича** — чистить нечего: загрузки фото в продукте ещё нет.
- **VLM-фолбэк со стелс-обвязкой** — см. §1.5, вместо него расширение-клиппер.
- FLUX-превью рендеров — под сомнением: text2img не рендерит по фото комнаты клиента,
  нерелевантный «черновик» антипродаёт кредиты. Сначала ручной прототип качества.

## 4. Слепые зоны (критик, полный список)

Дистрибуция (каналы РФ-дизайнеров не проработаны ни одной линзой); валидация спроса живыми
людьми до кода; недооценка слияния веток всеми оценками «effort»; браузерное расширение-клиппер
как бесплатный обход анти-бота; миграция localStorage → сервер как предусловие биллинга;
юридическая база (ИП, оферта, 152-ФЗ); авторские права на фото товаров магазинов в коммерческом
PDF клиенту (нерешённый вопрос ядрового сценария — проработать); операционный мониторинг
экстрактора (доля успешных извлечений, алерты деградации — обязательный слой при каскаде
из трёх нестабильных бесплатных провайдеров).
