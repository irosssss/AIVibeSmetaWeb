## 0. Терминология (важно не путать)

У дизайнеров это **разные документы**:
- **Смета** — перечень работ и черновых материалов (для бригады, до white box).
- **Ведомость** — отделочные материалы.
- **Спецификация / комплектация** — мебель, свет, декор = **FF&E**.

То, что строим мы, — это **спецификация/комплектация** (FF&E). «Смета» оставляем как разговорное/маркетинговое, но в проф-документе корректнее «спецификация».

## 1. Что у нас уже есть и это сильно (ров)

| Есть | Почему важно |
|---|---|
| **Детерминированная проверка эргономики** | Ни у одной FF&E-платформы (Studio Designer, DesignFiles, Fohlio, Gather, Houzz Pro) этого нет. |
| **Бюджет-оптимизатор value/₽** | Дизайнеры оптимизируют бюджет руками в Sheets; у CRM-конкурентов нет. |
| 3 варианта/категория (эконом/опт/премиум) | Де-факто встроенные аналоги-замены. |
| Старая цена/скидка, рейтинг/отзывы | Потребительский плюс, которого нет в классических FF&E-таблицах. |
| Бюджет-бар (подобрано/остаток/превышение) | Контроль бюджета на месте. |

## 2. Чего НЕ хватает до проф-уровня (главные пробелы)

**Поля позиции:** артикул/SKU, код-тэг (SO-01), **габариты Ш×Г×В**, материал/отделка/цвет, вес, единица измерения, количество с запасом (+5-15% на отделку), **фото-превью**, ссылка/QR на товар, помещение/зона.
**Экономика:** **наценка дизайнера (markup %)**, **две цены — trade (фабрика) и client (с наценкой)**, итоги по разделам, доставка/монтаж/налог отдельными строками.
**Процесс:** **статус закупки (8 стадий)** + даты, **версии сметы + согласование с клиентом**, **два режима — клиентский-вид vs рабочий** (скрывать наценку/trade от клиента), логистика (PO/Sidemark/Tracking).
**Экспорт:** **Excel/Google Sheets** с листами по 10-12 закупочным разделам (сейчас только PDF).

## 3. Бэклог (приоритеты)

### Сразу — высокий эффект, без блокеров (client-side)
1. **Наценка + две цены** (trade/client + markup%) + **итоги по разделам**. Ядро экономики дизайнера. (M)
2. ✅ **Два режима экспорта** (переключатель «Выгрузка: Рабочая / Для клиента» в смете): рабочая = себестоимость + наценка + цена клиента + бюджет; клиентская = только цена клиента (себестоимость/наценка/бюджет скрыты). Действует и в PDF (`web/pdf.js`), и в Excel (`web/xlsx.js`); файлы помечаются `-rabochaya` / `-klientu`. Снимает боль доверия. Проверено вживую на «Кирова». (M)
3. **Привязка позиции к комнате/зоне** + группировка сметы по помещениям. (S)
4. **Доставка/монтаж/налог отдельными строками** в итоге. (S)
5. **Единица измерения + запас на отделку** (+5-15%, коэффициент в нормах). (S)
6. **Lead time в неделях** по подтипу (quick ship 2-4 / каталог 6-10 / кастом 10-16 / импорт 16-24). (S)
7. **Импорт сметы из Excel** (round-trip по нашему шаблону колонок + гибкое сопоставление заголовков) — мост «не меняй инструмент», заносит готовую комплектацию в систему для эргономики/оптимизатора/двух режимов экспорта. *(зерно из внешнего совета 29.06)* (M)

### Требуют реального каталога фабрик
8. **Расширить позицию до FF&E-схемы**: SKU, бренд, габариты, материал/отделка, фото, вес. Фундамент. (M)
9. **Фото-превью + ссылка/QR на товар** в смете и PDF. (S)
10. **Аналоги в один клик с автопересчётом** сметы. Фундамент есть (варианты эконом/опт/премиум + бюджет-оптимизатор value/₽); нужен набор альтернатив по SKU из каталога фабрик. *(зерно из внешнего совета 29.06)* (M)
11. **Актуальность/свежесть цен**: дата прайса, флаг устаревания, обновление из фида. Без свежего фида смета протухает за неделю. *(зерно из внешнего совета 29.06)* (M)

### Структурное — превращает в рабочий инструмент
12. ✅ **Excel/Sheets-экспорт** (SheetJS, client-side) — `web/xlsx.js`. Книга: «Свод» (итоги по комнатам и разделам + бюджет), «Все позиции» (плоская мастер-таблица с автофильтром — сводная по любой оси) и лист на каждую комнату. Везде две цены (себестоимость/клиент). Вход в реальный процесс дизайнера. (L)
13. **Статус закупки (8 стадий) + даты** → живой трекер закупки, а не статичный список. (M)
14. **Версии сметы + согласование с клиентом** (snapshot, YDB). (L)

### Премиум / B2B
15. Техслой spec-sheet: гарантия, сертификаты, эко-маркировка, COM/COL+метраж. (M)
16. Логистика: PO number, Sidemark, Tracking. (S)

## 5. Чем обходим конкурентов в РФ
- Эргономика + бюджет-оптимизатор **внутри** сметы — ни у кого нет.
- Автоподстановка актуальных артикулов/цен/наличия из **каталога фабрик-партнёров** = те самые 40-100 часов ручной комплектации, которые дизайнер тратит сейчас. Реальный ров.
- AR/скан связывает позицию с геометрией комнаты (Room/Zone + Tag) — конкуренты дают только таблицу.
- Полностью РФ-стек (YandexGPT/GigaChat, YDB, ЮKassa, фабрики РФ); Studio Designer/DesignFiles/Houzz Pro в РФ недоступны/неудобны.
- Экспорт в формат, который дизайнер уже ждёт (Sheets/Excel по разделам) — мост, а не «смени инструмент».
