# Этап 0 «бумага + топливо»: пошаговый гайд владельца

Дата: 14.07.2026. Источники — официальные (yookassa.ru, yandex.cloud, nic.ru, справки Яндекс 360, доки Яндекс OAuth / VK ID, npd.nalog.ru); всё, что не удалось подтвердить первоисточником, помечено ⚠️. Собрано тремя research-агентами + юр-блок 152-ФЗ.

**Порядок действий (кратко):**
1. Статус: самозанятость сейчас → ИП при первых юрлицах-клиентах.
2. Уведомление Роскомнадзора (оператор ПДн) — ДО запуска сбора данных.
3. Реквизиты → страницы #policy / #offer (заменить плейсхолдеры `[УКАЖИТЕ …]`).
4. Yandex Cloud: аккаунт → зона Cloud DNS.
5. RU-CENTER: делегировать designledger.ru на `ns1/ns2.yandexcloud.net`.
6. Деплой + сертификат + домен на API Gateway.
7. Почта на домене (Яндекс 360) — нужна для ЮKassa/РКН/DMARC-эры.
8. OAuth: Яндекс ID (за вечер), VK ID (через VK Business ID — самозанятого хватает).
9. ЮKassa — подавать заявку ПОСЛЕДНЕЙ, когда сайт полностью готов (их модерация проверяет сайт).

## Шаг 1. Правовой статус: самозанятость vs ИП

**Рекомендация: начать самозанятым, перейти на ИП при первых клиентах-юрлицах или приближении к лимиту.**

Почему самозанятость (НПД) подходит для старта:
- Продавать доступ к СОБСТВЕННОМУ сервису (SaaS) на НПД **можно** — запрещена перепродажа чужого, не своё ПО (⚠️ подтверждено вторичными юр. источниками — письмо Минфина 15.03.2022 № 03-11-11/19322).
- Регистрация — приложение «Мой налог» / Госуслуги, минуты, бесплатно.
- Налог: 4% с платежей физлиц, 6% — юрлиц. **Никаких фиксированных взносов** (у ИП ~53 000 ₽/год независимо от дохода).
- ЮKassa работает с самозанятыми, **рекуррентные платежи (автосписание подписки) доступны** (официальный Q&A q20).
- VK Business ID (нужен для VK ID-входа) доступен самозанятым.

Ограничения самозанятости — когда переходить на ИП:
- **Платят только физлица** (карта). Клиент-студия со счётом юрлица — уже нельзя.
- Лимит дохода НПД — 2,4 млн ₽/год; лимит карт в ЮKassa — 500 000 ₽/мес.
- Комиссия ЮKassa выше: 3,5% + НДС против 2,8% у ИП.

Переход на ИП, когда понадобится:
- Онлайн через service.nalog.ru/gosreg (форма Р21001) или банки (Точка и др.) — **электронно бесплатно** (800 ₽ пошлины только за бумагу), 3 рабочих дня.
- ОКВЭД: основной **62.01** (разработка ПО), дополнительные 62.02, 62.09, **63.11** (обработка данных, размещение — классика SaaS).
- Налог: **УСН 6% «доходы»** — уведомление (форма 26.2-1) вместе с регистрацией или в 30 дней после.

## Шаг 2. Роскомнадзор: уведомление оператора ПДн (152-ФЗ, ст. 22)

Обязательно ДО начала реального сбора персональных данных (вход через Яндекс ID/VK ID = сбор имени/e-mail). С 2022 исключений почти не осталось; с 2025 действуют ощутимые штрафы за неподачу (для граждан 5–10 тыс. ₽, для ИП/юрлиц заметно больше).

- Форма: **pd.rkn.gov.ru** → «Уведомление об обработке персональных данных», подача электронно через Госуслуги, бесплатно.
- Что указывать: оператор (ты как самозанятый/ИП), цели (предоставление доступа к сервису, идентификация), категории данных (ФИО, e-mail, аватар — как в нашей политике #policy), категории субъектов (пользователи сервиса), правовые основания (согласие, договор-оферта), меры защиты (HTTPS, авторизация через Яндекс ID/VK ID), **местонахождение БД — РФ** (Yandex Cloud), трансграничной передачи нет.
- Наша страница #policy уже структурирована под эти же пункты — можно переписывать из неё.

## Шаг 4. Yandex Cloud: аккаунт и зона DNS

1. Войти консоль cloud.yandex.ru через Яндекс ID → создать **платёжный аккаунт**:
   - Сейчас — «Физическое лицо» (карта обязательна). Грант новичку **≥ 4 000 ₽** (срок действия — смотреть в биллинге; ⚠️ исторически ~60 дней).
   - **Тип аккаунта потом не меняется** — при переходе на ИП создаётся НОВЫЙ бизнес-аккаунт и облако перепривязывается (без простоя). ИП на бизнес-аккаунте получает акты/УПД для бухгалтерии; физлицу — только чеки. Грант второй раз не дают.
2. **Сразу создать публичную DNS-зону** `designledger.ru.` в Cloud DNS (консоль → Cloud DNS → создать зону). ВАЖНО: зону создать ДО делегирования — Cloud DNS не проверяет владение доменом, «пустую» делегацию может занять чужой.
3. Перенести в зону записи, которые уже есть у RU-CENTER (если есть), + подготовить записи почты (шаг 7).

Free tier serverless (в месяц): Functions 1 млн вызовов, API Gateway 100 тыс. запросов, YDB 1 млн RU + 1 ГБ, Object Storage 1 ГБ, 100 ГБ трафика. **Прототипу с десятками пользователей — около нуля рублей**; платными будут только DNS-зона (десятки ₽/мес) и токены YandexGPT.

YandexGPT: сервис теперь называется **Yandex AI Studio** (доки foundation-models редиректят на aistudio.yandex.ru). Цены (синхрон, за 1000 токенов, с НДС): **Lite 0,2 ₽**, Pro 5.1 — 0,8 ₽; асинхрон вдвое дешевле. Каскад клиппера на Lite: запрос ≈ 0,5 коп., 1000 запросов/день ≈ 150 ₽/мес. Ключ: сервисный аккаунт → API-ключ (scope `yc.ai.languageModels.execute`), роль `ai.languageModels.user`.

## Шаг 5. RU-CENTER: делегирование домена

В ЛК nic.ru:
1. Сначала проверить два орг-момента: **тариф домена** (бесплатный редактор DNS «DNS-премиум» входит только в тарифы «Оптимальный»+; на «Базовом» — нет, но он и не понадобится после делегирования) и **цену продления** (за 60 дней до конца можно продлить; ⚠️ жалобы 2025 на рост цен продления — проверить и при желании перенести домен к другому регистратору потом). С **01.09.2026** RU-CENTER вводит обязательную идентификацию через Госуслуги (ЕСИА) — пройти заранее.
2. Главная ЛК → **«Домены»** → напротив designledger.ru столбец **«DNS»** → «Изменить»/«Делегировать» → «Указать DNS-серверы самостоятельно» → вписать:
   - `ns1.yandexcloud.net.`
   - `ns2.yandexcloud.net.`
3. Сохранить. Обновление для .ru — **от 2 до 24 часов**. После этого все записи ведутся в Cloud DNS, редактор Руцентра больше ни на что не влияет.
4. Поддомены (`app.`, `api.`) — просто записи в зоне Cloud DNS, отдельно «заводить» не нужно.

## Шаг 6. Деплой и HTTPS

Рекомендуемая топология: **API Gateway — единая точка** (и статика из Object Storage, и `/api/*` на Cloud Functions): один домен, один сертификат, нет CORS.

1. CLI: `yc init`. Сервисный аккаунт + роли: `functions.functionInvoker`, `ydb.editor`, `storage.viewer`, (для LLM) `ai.languageModels.user`.
2. YDB serverless: `yc ydb database create designledger --serverless`.
3. Функция портала (код уже в репо, PR #15): runtime **nodejs22**.
4. Статика: `npx vite build` → бакет Object Storage.
5. API Gateway с OpenAPI-спекой (интеграции object_storage + cloud_functions).
6. Certificate Manager → Let's Encrypt-сертификат на `designledger.ru` (DNS-challenge — TXT-запись в нашей же зоне, автопродление каждые 90 дней).
7. Привязать домен к шлюзу: для голого `designledger.ru` — **ANAME-запись** в Cloud DNS (CNAME на голый домен нельзя по RFC); `app.` при желании — CNAME на служебный домен шлюза.

## Шаг 7. Почта на домене (Яндекс 360)

Нужна: контактный e-mail для ЮKassa-анкеты, РКН, юр-страниц, писем клиентам.

- ⚠️ Бесплатный тариф Яндекс 360 официально свернули в 2023, оставив организациям **до 3 пользователей**; на витрине тарифов его нет — проверять прямо в мастере admin.yandex.ru при подключении. Платный «Минимальный» — 319 ₽/чел/мес (**с 01.07.2026 индексация до ~485 ₽**; с 29.06.2026 IMAP/SMTP — только на платных).
- Подключение: admin.yandex.ru → добавить домен → подтвердить владение **TXT-записью** (`yandex-verification: …`) → добавить в Cloud DNS записи:
  - MX: host `@`, значение `mx.yandex.net.` (с точкой!), приоритет 10;
  - SPF (TXT, `@`): `v=spf1 redirect=_spf.yandex.net`;
  - DKIM (TXT, `mail._domainkey`): значение из админки 360.
- Завести ящики: `hello@designledger.ru` (общий/ЮKassa), `legal@` (обращения по ПДн — вписать в #policy).

## Шаг 8. OAuth-приложения

**Яндекс ID** (за вечер, модерации нет):
- oauth.yandex.ru → создать приложение типа «авторизация пользователей на стороннем сервисе», платформа «Веб-сервисы».
- Redirect URI: прод-callback на designledger.ru.
- Права: `login:email`, `login:info`, `login:avatar` (ровно то, что заявлено в #policy).
- Заполнить имя/иконку/описание — их видит пользователь в окне согласия. Тип приложения потом не меняется.

**VK ID** (планировать заранее — нужен статус):
- Старый OAuth ВК не работает; текущий — VK ID (OAuth 2.1), вход аккаунтами VK/OK/Mail.
- Кабинет id.vk.ru → «Мои приложения» → Web: базовый домен `designledger.ru` + **доверенный Redirect URL строго на том же домене**. Получаешь Client ID / client_secret / сервисный ключ.
- **Обязательная верификация через VK Business ID** (доступен юрлицам, ИП и **самозанятым**) — новым приложениям в течение 60 дней, иначе блок. Ещё один аргумент оформить самозанятость первым шагом.
- E-mail пользователя VK отдаёт опционально — в коде предусмотреть вход без e-mail.
