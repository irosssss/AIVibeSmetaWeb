# Веб-прототип AIVibe — кабинет дизайнера (оживление 2026-06)

> Рабочий артефакт. Фиксирует состояние клиентского веб-прототипа и план его
> «оживления» в реальный продукт. Создан в сессии 2026-06-26 после пивота на
> стейджинг/дизайнеров (см. `STRATEGY_SCENARIOS_2026-06-24.md`, `MARKET_STAGING_RU_2026-06.md`).

## 1. Что это и где лежит

Кликабельный **веб-прототип личного кабинета AIVibe** на React (через Babel-standalone
в браузере, без сборки). Это **витрина + рабочий экран дизайнера**, не iOS-приложение.

- **Расположение:** `~/Downloads/web 3v ai/` (вне основного репозитория — в публичный репозиторий пока НЕ вносился).
- **Стек:** React 18 (UMD) + Babel-standalone, чистый CSS (`styles.css` + `app.css`), без npm-сборки.
- **Точка входа:** `AIVibe.html` (и копия `index.html` для отдачи по корню сервера).
- **Данные:** единый mock-слой `mock.js` (`window.AIVibeAPI`) + `project-data.js` (детали проекта). Тел методов помечены `// → API:` под будущую замену на Yandex Cloud Functions + YDB — UI переписывать не придётся.

## 2. Что сделано в этой сессии (re-skin под дизайнера)

Прототип изначально продавал **старый курс** (LiDAR-скан, AR, потребитель, маркетплейс
Ozon/Wildberries, «Скачать в App Store»). Переодет под новый: **смета + проверка норм для
дизайнера интерьера, веб-first, каталог фабрик-партнёров.**

- **Промо-лицо:** герой «Спецификация, которая сходится сама»; CTA «Собрать смету» (вместо App Store); шаги Габариты → Смета и нормы → 3 бюджета; бенто Смета · AI-дизайнер · Проверка эргономики · 3 бюджета; футер «Соберите смету сегодня».
- **GitHub удалён** полностью (блок, пункт в nav, ссылка в футере) — по запросу владельца.
- **Маркетплейс вычищен везде** (правило `CLAUDE.md`: Ozon/WB недопустимы): нет ни одного «Ozon/Wildberries/маркетплейс/корзина» в коде. «Корзина» → «смета», «скидки маркетплейсов» → «скидка по каталогу».
- **Каталог:** товарные источники `mp:"wb"/"ozon"` → коды фабрик `"f1"/"f2"`, id `wb-`/`oz-` → `f1-`/`f2-`. Бейджи перекрашены (CSS `.mp.f1`/`.mp.f2`, тёплый дуб / шалфей).
- **Кабинет:** профиль (KPI «Смет собрано» вместо «AR-сканов»), настройки («Подбор по каталогу фабрик», «Автопроверка норм»), модалка «Новый проект» (Габариты → анализ → смета), избранное/шоп-лист с фабриками.
- **AR/LiDAR-следы** в тексте убраны: «скан RoomPlan 2 · LiDAR» → «план комнаты», кнопка «Примерить в AR» → «Выгрузить PDF», новость про AR → про проверку эргономики. (AR остаётся апселлом будущего, не стартовым каналом.)

Проверено вживую: рендер без ошибок консоли; на промо и в экране проекта — 0 упоминаний маркетплейса; бейджи фабрик «Дубрава»/«Линея» отображаются.

## 3. Как запустить локально
```bash
# из папки прототипа (Babel грузит .jsx по сети — нужен HTTP-сервер, file:// не сработает)
python3 -m http.server 8753 --directory "~/Downloads/web 3v ai"
# открыть http://localhost:8753/
```
Внизу слева — свитчер прототипа: **Промо / Кабинет / Админка**.

## 4. Архитектура данных (мост к реальному бэкенду)

`mock.js` намеренно повторяет сигнатуры будущего API. Точки замены (тело мока → `fetch`):

| Метод мока | Будущий эндпоинт |
|---|---|
| `auth.loginWithYandex/VK`, `getSession`, `logout` | `POST /api/auth/oauth/{provider}`, `GET/POST /api/auth/session` |
| `profile.get`, `profile.analytics` | `GET /api/profile`, `GET /api/profile/analytics` |
| `projects.list`, `summary`, `get(id)`, `chat(id,msg)` | `GET /api/projects…`, AI-чат через ai-advisor |
| `favorites.list/remove` | `GET/DELETE /api/favorites` |
| `news.*`, `users.*`, `analytics.overview` | CRUD новостей, админ-пользователи, метрики |

## 5. Заглушки и договорённости (важно для честности)

- **Фабрики «Дубрава» / «Линея» — выдуманные плейсхолдеры** (коды `f1`/`f2`). Реальных договорённостей нет; перед публикацией заменить на настоящий фид фабрик-партнёров.
- **Цены, наличие, сроки поставки, скидки — фейковые** (детерминированный хеш от id). Реальные — из каталога фабрик в YDB.
- **AI-чат** отвечает по ключевым словам из `project-data.js`, без реального YandexGPT.
- Всё клиентское, **секретов в коде нет** — поэтому работать можно открыто.

## 6. План оживления (от дешёвого к дорогому)

Принцип: автоматизируем интерфейс + дешёвую кнопку-смету (наш ров, без GPU) сразу;
дорогую коробку (нейро-стейджинг) фейкаем в режиме «Волшебник из Оз» до первых оплат.

1. **Рабочий экран сметы → реальный бэкенд.** Заменить мок `projects.get/chat` на Yandex Cloud Functions + движок эргономики (ArrangementEngine + DesignNorms). Это и есть автоматизация на рве.
2. **Выгрузка спецификации** (PDF/Excel) — продающий артефакт дизайнера (сейчас кнопка-заглушка «Выгрузить PDF»).
3. **Авторизация** Яндекс ID / VK ID (моки уже под это заложены).
4. **Оплата ЮKassa** на сайте (Apple IAP в РФ мёртв; контур — на вебе). ИП есть.
5. **Реальный фид фабрик** вместо плейсхолдеров `f1`/`f2`.
6. **Стейджинг (фото до/после)** — позже, отдельной кнопкой, сначала вручную/внешним API.
7. **iOS/AR** — апселл-премиум, не стартовый канал. Android ещё позже.
