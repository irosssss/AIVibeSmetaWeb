# Design Ledger — витрина дизайн-системы (для Claude Design)

Самодостаточные HTML-превью канона **«warm editorial Atelier»** — для заливки в
проект claude.ai/design через скилл `/design-sync` (инструмент `DesignSync`).

## Важно: это ЗЕРКАЛО, а не источник правды

Источник канона — `web/styles.css` (`:root`-токены). Здесь токены **скопированы
инлайном**, чтобы каждая карточка рендерилась standalone в панели Design System.
Меняешь канон — правишь `styles.css`, потом синхронно обновляешь превью тут.
(SOUL: один источник правды; витрина отражает, не подменяет.)

## Формат карточки

Каждый `*.html` — первая строка `<!-- @dsCard group="…" -->` (панель группирует
по `group`), дальше самодостаточный документ с инлайн-стилями. Явная регистрация
карточек не нужна — приложение строит индекс из этих маркеров.

## Что внутри

- `foundations/` — Основы: палитра, типографика, радиусы/тени.
- `components/` — Компоненты: кнопки, eyebrow, спец-таблица «две цены»,
  KPI-карточка, статус-чипы, карточка товара.

## Шрифты

Канон — Spectral (display) / Inter (body) / JetBrains Mono. В standalone-превью
без webfont'ов подхватывается фолбэк (Georgia / system-ui / mono) — как в
`styles.css`. Вживую в приложении грузятся настоящие через `@fontsource`.

## Как залить в Claude Design

1. `/design-sync` (нужен claude.ai-логин с design-скоупами; иначе `/design-login`).
2. Выбрать/создать проект типа design-system.
3. Скилл соберёт план (какие пути пишем), покажет на подтверждение, зальёт
   инкрементально. Внешний эффект — руками владельца.
